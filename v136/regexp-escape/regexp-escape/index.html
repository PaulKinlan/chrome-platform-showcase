      leading prefix guard

      When the first character is an ASCII digit or letter, RegExp.escape() uses \xHH. That keeps the escaped fragment safe when a regex builder concatenates it after \0, \1, or \c.

      the code

      function highlightUserInput(haystack, userInput) {
  // Pre-136: bring your own escapeRegExp().
  // 136+: ship the standard one:
  const safe = RegExp.escape(userInput);
  const re = new RegExp(safe, "g");
  return haystack.replace(re, m => `<mark>${m}</mark>`);
}
