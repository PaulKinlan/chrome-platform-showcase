new RegExp(input) — metasymbols break it
If the user types version(1.0), new RegExp("version(1.0)") creates a group (1.0) where . matches any character. The search matches version110, versionX1Y0, anything that fits the regex. Literal text is no longer literal.
RegExp.escape() — safe literal matching
new RegExp(RegExp.escape(input)) escapes the parens and dot: the pattern becomes version\(1\.0\). Only the exact string version(1.0) matches. Every regex metasymbol the user might type — . * + ? ( ) [ ] { } ^ $ | \ — is rendered literal.
function buildPattern(input, opts) {
  const escaped = opts.regex
    ? input               // user wants real regex
    : RegExp.escape(input); // literal match — safe from user input

  const flags = 'g' + (opts.case ? 'i' : '');
  const source = opts.word
    ? `\\b${escaped}\\b`  // whole-word boundary
    : escaped;

  return new RegExp(source, flags);
}
