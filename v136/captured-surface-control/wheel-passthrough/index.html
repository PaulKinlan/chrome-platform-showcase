Wheel passthrough

The video-conferencing pattern that motivated the spec: a live "preview tile" of the captured tab where the user can wheel/scroll inside the tile and have those wheel deltas forwarded to the captured surface — without having to leave the call to do it.

the code

const tile = document.getElementById("tile");
const ctrl = new CaptureController();
const stream = await navigator.mediaDevices.getDisplayMedia({ controller: ctrl });
tile.querySelector("video").srcObject = stream;

tile.addEventListener("wheel", async (e) => {
  e.preventDefault(); // don't scroll the tile itself
  try {
    await ctrl.forwardWheel({ deltaX: e.deltaX, deltaY: e.deltaY });
  } catch (err) {
    // Permission denied or surface unsupported.
  }
}, { passive: false });

why this angle

The sibling concept exposes the API surface — "press the up/down buttons, they call forwardWheel." Real VC products don't ship those buttons; users scroll the preview tile with their real trackpad or mouse wheel. The DOM event → CSC translation is what the spec was designed for, and is what makes the difference between "demo" and "shippable feature." This page wires the actual wheel handler.
