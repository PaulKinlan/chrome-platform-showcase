State Machine Visualiser

An AudioContext can be in one of five states — suspended, running, closed, and (new in v136) interrupted. Drive every legal transition from the buttons below, watch the highlighted node move, and read the timestamped statechange log.

Note: the interrupted state is only reachable when the OS or browser pulls audio focus (phone call, screen lock, another exclusive-audio app). On desktop you may only see suspended → running → closed. The diagram still lights up the states that are reachable.

legal transitions

  • suspended → running via resume()
  • running → suspended via suspend() or user pause
  • running → interrupted when the OS pulls audio focus
  • interrupted → running when focus returns (auto or explicit resume())
  • any state → closed via close()
