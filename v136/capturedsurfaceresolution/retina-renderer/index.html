Retina Renderer

High-DPR remote desktop scenario: use the captured surface's resolution and device pixel ratio to render the remote surface at true 1:1 pixel density. Shows the math behind scaling a capture stream to a hi-DPI canvas — and what blurry vs sharp looks like when you do or don't account for DPR.

When capturing a Retina screen, the OS delivers 2× (or 3×) pixels. Without CapturedSurfaceResolution, you could only see the CSS (logical) pixel dimensions. Chrome 136's new API exposes the physical pixel dimensions, letting remote-desktop apps size their canvas at 1:1 device pixels instead of blurry 2× upscale.
