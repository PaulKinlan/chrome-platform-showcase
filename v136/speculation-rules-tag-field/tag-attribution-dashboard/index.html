Build multiple speculation rule sets, each with a different tag. The dashboard shows the Sec-Speculation-Tags header value each URL would carry, attributes speculative navigation hits/misses per tag, and generates the JSON ready to embed in a <script type="speculationrules"> block.

checking SpeculationRules support…
Chrome 136 adds an optional tag field to each rule set. The value is sent as the Sec-Speculation-Tags request header when Chrome performs a speculative prefetch or prerender. This lets CDN edge workers, server-side analytics, or A/B testing frameworks distinguish which rule source triggered the request.
