Type-agnostic var()

var() fallback values no longer have to match the variable's registered type. Lets authors mix typed and untyped fallbacks in the same expression.

The canonical shape of the API:

.btn { color: var(--accent, currentColor); }
/* The fallback no longer has to be the same registered
   type as the variable. */
