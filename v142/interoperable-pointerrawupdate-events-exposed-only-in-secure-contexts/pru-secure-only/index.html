Secure-context Only

pointerrawupdate is now restricted to secure contexts across browsers. Removes a cross-browser inconsistency where Chrome would deliver the event on http while others didn't.

The canonical shape of the API:

// pointerrawupdate only fires on secure (https/localhost)
// contexts; on http it never delivers. Brings Chrome
// in line with Safari/Firefox.
el.addEventListener("pointerrawupdate", e => {
  // sub-frame fidelity input data
});
