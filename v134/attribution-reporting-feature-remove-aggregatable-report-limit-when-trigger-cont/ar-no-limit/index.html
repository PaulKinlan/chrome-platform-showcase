Aggregatable Report Limit Lifted

When a trigger registration sets trigger_context_id, the per-source aggregatable report cap (previously 20) no longer applies. Each conversion can spawn its own report. Use the simulator below to see how report yield changes when the cap is gone.

Heads up Attribution Reporting is a Privacy Sandbox API. To actually exercise the network behaviour, enable chrome://flags/#privacy-sandbox-ads-apis-override and register with attributionsrc-bearing pixels. The simulator below models the limit behaviour client-side so you can see what changed.

The code

// reporting origin sends, on a trigger pixel:
Attribution-Reporting-Register-Trigger: {
  "aggregatable_trigger_data": [...],
  "aggregatable_values": {"key": 1},
  "trigger_context_id": "conv_42"   // <-- non-null lifts the cap
}
