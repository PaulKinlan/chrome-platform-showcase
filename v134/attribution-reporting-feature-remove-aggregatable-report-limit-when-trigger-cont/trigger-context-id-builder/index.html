Attribution Reporting: server-side API — simulation mode
Before Chrome 134: every source could generate at most 20 aggregatable reports regardless of trigger volume. If you sent 50 conversions for the same source, 30 were silently dropped. Chrome 134 lifts that cap when trigger_context_id is set on the trigger registration, enabling unlimited aggregatable reports per source for high-volume flows.
