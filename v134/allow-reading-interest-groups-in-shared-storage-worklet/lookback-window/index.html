Reach measurement with a lookback window

The driving use case for adding interestGroups() to Shared Storage worklets was reach measurement: how many users in audience X have been served by ad campaign Y in the last N days. The lookbackWindow property on each IG is what makes that possible without dragging along arbitrary user history. Move the slider to see how the worklet would filter the audience before sending a single aggregate to the reporting endpoint.

This API is only exposed inside a SharedStorageWorkletGlobalScope in a fenced frame or window context with the shared-storage permission policy. The demo simulates the worklet locally so you can interact with the algorithm; the production code listing below is what runs inside Chrome 134+.

The worklet code

// reach-worklet.js — runs inside Shared Storage worklet
class Reach {
  async run({ campaign, maxAgeMs }) {
    const igs = await interestGroups();
    const reached = igs.filter((g) =>
      g.lookbackWindow < maxAgeMs &&
      (!campaign || g.userBiddingSignals?.tags?.includes(campaign))
    );
    if (reached.length) {
      privateAggregation.contributeToHistogram({
        bucket: BigInt(campaignBucket(campaign)),
        value: 1n,
      });
    }
  }
}
register('reach', Reach);
