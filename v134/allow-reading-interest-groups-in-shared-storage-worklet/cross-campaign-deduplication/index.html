Cross-campaign Deduplication

A Shared Storage worklet that reads interestGroups() can deduplicate reach across campaigns — avoiding double-counting users who saw multiple campaigns. Simulate a worklet run across a mock user population.

checking Shared Storage…
Without interestGroups() in the worklet, you can't tell which users are in multiple campaigns — so your reach count overcounts. The worklet can now read all IGs for the origin, deduplicate across campaigns, and contribute a single aggregate count.

// Shared Storage worklet (Chrome 134+) class ReachMeasurementOperation { async run(data, { sharedStorage }) { const campaign = data.campaign; // New in Chrome 134: read interest groups const igs = await sharedStorage.interestGroups(); // Deduplicate: only count if not already seen const seen = await sharedStorage.get('seen_' + campaign); if (seen) return; // already counted — no double-count // Check if user is in target audience const inAudience = igs.some(ig => ig.name.startsWith(campaign.split('-')[0]) ); if (inAudience) { await sharedStorage.set('seen_' + campaign, '1'); privateAggregation.contributeToHistogram({ bucket: BigInt(data.bucketOffset), value: 1 }); } } } register('reach-measurement', ReachMeasurementOperation);
