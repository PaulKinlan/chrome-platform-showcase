Round-trip savings on a header-bidding page

The header-bidding tag market’s longstanding pattern: a single ad slot can have 6–10 SSPs (sellers) bidding, each historically calling navigator.getInterestGroupAdAuctionData() separately and shipping its own encrypted payload to its own Bidding & Auction service. The Chrome 134 change accepts multiple seller configurations in one call. Pick a seller count and watch the network timeline collapse.

probing navigator.getInterestGroupAdAuctionData multi-seller…
Protected Audience B&A only runs on real ad infrastructure. The model here applies the call-count and encryption-cost numbers from the explainer to a configurable seller count so you can see the saving shape.

The code

// before (still works):
for (const seller of sellers) {
  const payload = await navigator.getInterestGroupAdAuctionData({ seller });
  send(seller, payload.request);
}

// Chrome 134+: one call, payload per seller
const payloads = await navigator.getInterestGroupAdAuctionData({
  sellers: sellers,  // array
});
for (const seller of sellers) send(seller, payloads.requests[seller]);
