Before Chrome 134, each seller needed a separate getInterestGroupAdAuctionData() call — one network round-trip per SSP. Chrome 134 accepts an array of seller configs in a single call and returns one encrypted payload per seller, allowing multiple SSPs sharing a page's JS to run their B&A auctions from a single fetch.
