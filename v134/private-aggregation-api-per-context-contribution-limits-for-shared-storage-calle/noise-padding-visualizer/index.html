why this is two-sided

    
Two complaints landed on the spec:

    
    
      
  1. "20 isn't enough" — a fraud-detection or attribution worklet that wants 50+ buckets per report used to have to split work across multiple shared-storage operations.
    2. 
      
  2. "20 is wasteful" — a simple reach-measurement worklet that contributes only 1 bucket still paid the noise-padding cost of 20 slots, inflating network and aggregation-service compute.
    3. 
    

    
v134 lets the worklet pick. A modest cap = small report, small padding. A generous cap = no overflow, but bigger reports.
