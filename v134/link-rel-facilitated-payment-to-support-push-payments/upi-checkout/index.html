UPI / PIX checkout

The WICG explainer’s motivating market: India’s UPI and Brazil’s PIX, where checkout is a push — the merchant publishes a payment URI, the user opens their bank app, scans/taps, and money is pushed back to the merchant. <link rel="facilitated-payment"> turns that into a declarative signal so Chrome (and the user’s installed bank app) can offer to handle it without copy-paste. Configure a cart below, hit publish, and the page injects a real link tag while emulating the wallet round-trip.

probing link rel=facilitated-payment…
The behaviour requires a paired Android wallet client; on desktop the link is parsed but no client surfaces. The demo injects a real <link rel="facilitated-payment"> with the same URI structure Google Pay / PhonePe / PayPal Brazil use.
