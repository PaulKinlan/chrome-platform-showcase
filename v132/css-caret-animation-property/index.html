CSS caret-animation property

Chromium supports animation of the caret-color property, but when animated the default blinking behavior of the caret interferes with the animation. For instance, see the example at https://drafts.csswg.org/css-ui/#caret-animation where an animation from blue to red and back is rendered as a blinking cursor that is randomly blue or red.     The naive version always animates. The respectful version turns the animation off under prefers-reduced-motion: reduce and hands the blink back to the system.

  5. Chat Input

    A chat interface where the caret changes per input mode: blink for normal text, none (steady) for code snippets, coloured blink for @mention mode. Context-sensitive caret signalling — pure CSS, no JavaScript.

  6. Code Editor

    An editor mockup with four Vim-style modes — Normal (steady grey), Insert (green blink), Visual (blue steady), Search (red blink). Each mode sets caret-color + caret-animation via a data-mode attribute CSS selector.

why it shipped

When the caret-color is animated the default blinking cursor interferes with the animation. This CSS property enables authors to disable the blinking when animating the caret color.
