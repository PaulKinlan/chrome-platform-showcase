ar_debug cookie change

Attribution Reporting relaxes the ar_debug cookie requirement when partitioning provides equivalent privacy. Less ceremony to enable debug output.

The canonical shape of the API:

# AR debug headers no longer require the ar_debug=1 cookie
# in third-party contexts where partitioning protects the
# user — easier to enable verbose attribution logging.
