Worklet Simulator

Chrome 132 adds a savedQuery option to sharedStorage.selectURL(). On the first call with a given query name, two budget units are charged. On subsequent calls with the same query name in the same page load, the result is returned from cache — no further budget is consumed. This simulator shows exactly when charges are applied versus when they are free-from-cache.

// Chrome 132: savedQuery parameter
const result = await sharedStorage.selectURL(
  'my-worklet',
  urls,
  {
    savedQuery: 'my-query-name',   // ← NEW in Chrome 132
    keepAlive: true,
  }
);

// First call with 'my-query-name':
//   → Runs worklet, charges 2 bits from page-load budget
//   → Saves result under query name

// Second call with same 'my-query-name' (same page load):
//   → Returns cached result immediately
//   → No budget charged ✓

// Without savedQuery (legacy):
//   → Every call charges 2 bits
//   → 6 ad slots × 2 bits = 12 bits — exceeds per-page budget
