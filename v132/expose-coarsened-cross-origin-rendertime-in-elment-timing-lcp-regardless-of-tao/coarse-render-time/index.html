Coarsened renderTime in element-timing / LCP

A real LCP image is loaded from a cross-origin host with NO Timing-Allow-Origin header. The PerformanceObserver below shows the renderTime that Chrome reports. Pre-132 it would have been 0; in 132+ it is non-zero, coarsened to a 4 ms multiple.

attaching observer…
fieldvalue (ms)note
startTime—fetch began
renderTime—coarsened (4ms multiple) when cross-origin
loadTime—image bytes downloaded
identifier—elementtiming attribute

the observer

new PerformanceObserver(list => {
  for (const e of list.getEntries()) {
    console.log(e.name, e.renderTime, e.startTime); // renderTime non-zero even without TAO
  }
}).observe({ type: "largest-contentful-paint", buffered: true });
