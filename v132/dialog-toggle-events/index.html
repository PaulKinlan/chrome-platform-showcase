concepts

  1. Dialog ToggleEvent

    <dialog> fires the same toggle/ToggleEvent that <details> and popovers do. Single listener pattern for any 'is this open?' state.

  2. cancel before close

    Veto a dismissal from beforetoggle with preventDefault() — unsaved-changes guard works against Esc, backdrop, and form submit.

  3. dialog state machine

    Visualise the closed ↔ open state machine. Click show(), showModal(), close() and watch the diagram light up while every event logs with oldState and newState.

  4. CSS animation on toggle

    Three presets — fade, slide, spring — that animate dialog entrance and exit using only @starting-style, transition: overlay, and the new toggle events. Zero JS choreography.
