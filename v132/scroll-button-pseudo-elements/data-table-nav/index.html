Data Table Nav

A wide data table with ::scroll-button(inline-start) and ::scroll-button(inline-end) arrows for accessible horizontal navigation. The buttons are keyboard-focusable and apply the :disabled state automatically when the table is at its scroll edges — all without any JavaScript click handlers.

Pre-132 vs Chrome 132+

Pre-Chrome 132
To add left/right scroll buttons to a data table, authors had to add real <button> elements to the DOM, wire up click event handlers that called scrollBy(), and manually track scroll position to disable the buttons at the edges using another event listener on scroll. Each table needed its own wiring.
Chrome 132+
::scroll-button(inline-start) and ::scroll-button(inline-end) on the scroll container generate keyboard-accessible scroll buttons in CSS. The browser applies :disabled automatically at the scroll edges. No DOM elements to add, no event listeners, no manual position tracking.
/*
  ::scroll-button() — CSS-generated scroll navigation buttons
  No JS, no extra DOM elements, no event listeners
*/

.table-scroller {
  overflow-x: auto;
  display: flex;          /* puts buttons inline with scroll area */
  align-items: stretch;
}

/* Left / right navigation arrows */
.table-scroller::scroll-button(inline-start) {
  content: "←";
  width: 2rem;
  background: var(--bg-stone);
  border-right: 1.5px solid var(--border-black);
  cursor: pointer;
  font-size: 0.9rem;
}

.table-scroller::scroll-button(inline-end) {
  content: "→";
  width: 2rem;
  background: var(--bg-stone);
  border-left: 1.5px solid var(--border-black);
  cursor: pointer;
}

/* Browser auto-applies :disabled when no more scroll room */
.table-scroller::scroll-button(inline-start):disabled,
.table-scroller::scroll-button(inline-end):disabled {
  color: var(--text-muted);
  cursor: not-allowed;
}
