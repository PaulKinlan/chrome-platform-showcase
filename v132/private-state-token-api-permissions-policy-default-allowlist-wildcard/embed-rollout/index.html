Why the default flipped: the fraud-detection embed rollout

Before 132 the default Permissions-Policy for PST issuance & redemption was self. A publisher embedding a third-party fraud-detection vendor had to ship a header override for every page, or the vendor's widget silently failed. v132 flips the default to *; the publisher only has to set the header when they want to restrict.

pre-132 publisher rollout

132+ publisher rollout

headers pre-132

HTTP/1.1 200 OK
Permissions-Policy: private-state-token-issuance=(self "https://fraud.example"),
                    private-state-token-redemption=(self "https://fraud.example")
# every domain the publisher embeds has to be listed here

headers 132+

HTTP/1.1 200 OK
# nothing - default '*' lets every iframe call PST
# Permissions-Policy header only required to LOCK DOWN
The wildcard default mirrors how attestation cookies historically worked for fraud vendors. Publishers stay out of the loop; the browser still enforces PST issuer attestation lists.
