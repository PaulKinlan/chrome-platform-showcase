Freeze Timeline

Background tabs on Android freeze sooner — within ~1 second of backgrounding, down from 5. Save more aggressively on the freeze event; opt into Background Sync where you need work to continue.

The canonical shape of the API:

// No API change — Chrome on Android now freezes
// background tabs faster (5s → 1s after backgrounding).
// Effects: lower battery drain, but sites doing
// background sync should opt into the
// page-lifecycle hooks to keep tasks running.

document.addEventListener("freeze", () => {
  // last chance to persist state
});
