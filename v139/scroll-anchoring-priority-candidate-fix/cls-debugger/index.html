how Chrome 139 fixes this

Before v139, the scroll anchor candidate selection algorithm could pick an element below the first visible line, causing a scroll adjustment that pushed earlier content out of view. Chrome 139 gives priority to candidates that appear earlier in the viewport — so when content is inserted above the fold, the anchor is still the element you were reading, not one that happens to be deeper in the DOM.

/* No CSS change required — v139 improves the algorithm
   that selects overflow-anchor candidates.
   Previously: later in-viewport elements could win.
   Now:        earlier in-viewport elements take priority.

   You can still opt out per-element:
   overflow-anchor: none; */
