Anchor Fix

Chromes existing scroll anchoring now prefers earlier in-viewport candidates over later ones when picking the anchor element. Layout shifts caused by upstream insertions become less disruptive.

The canonical shape of the API:

/* No new CSS; Chromes scroll-anchoring algorithm
   now picks a more stable anchor when content above
   the viewport changes (lazy-loaded images, inserted
   ads, hydration), reducing layout shift jumps. */
