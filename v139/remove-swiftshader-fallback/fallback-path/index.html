Fallback Path

SwiftShader — Chromes software-rendered WebGL fallback — is removed. Devices that can't run GPU-backed WebGL now get null from getContext('webgl'); sites should branch on that.

The canonical shape of the API:

# Chrome no longer falls back to SwiftShader
# (software WebGL renderer) when hardware is unavailable.
# Sites should check WebGLRenderingContext support
# at runtime and offer a non-3D fallback.

const gl = document.createElement("canvas").getContext("webgl");
if (!gl) showFallbackContent();
