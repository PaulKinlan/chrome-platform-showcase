Graceful degradation when WebGL is unavailable

  1. Feature-detect before initialising: const gl = canvas.getContext('webgl2') ?? canvas.getContext('webgl');
  2. If gl === null, fall back to Canvas 2D: canvas.getContext('2d') — always available, hardware-accelerated via Skia on most platforms.
  3. For charts and data visualisation: SVG renders without GPU at full fidelity. Consider D3 or Observable Plot which support SVG output natively.
  4. For 3D scenes where geometry is simple: CSS 3D transforms are GPU-accelerated via the compositor even when WebGL is unavailable.
  5. For complex 3D: show a static pre-rendered image with a "your browser does not support WebGL" message — never a blank page.
  6. Check chrome://gpu to confirm hardware acceleration status and diagnose blocklisted drivers.
