awaiting request build
  1. Populate SecurePaymentConfirmationRequest with credential IDs, challenge, RP ID, instrument details, payee fields, timeout, and optional payment entity logos.
  2. Construct new PaymentRequest([{ supportedMethods: "secure-payment-confirmation", data }], details).
  3. Call canMakePayment() to decide whether to offer SPC or immediately show merchant fallback authentication.
  4. Handle separate merchant outcomes: success, continue without SPC, cancellation, authentication failure, or constructor validation errors.
