Port Allocation Simulator

Windows Chromium switches from sequential to randomised TCP source ports, matching macOS/Linux. Sequential allocation lets off-path attackers predict the next 5-tuple. Click below to compare a burst of 32 allocations.

Sequential ports let an off-path attacker spray RST or SYN at predictable 5-tuples. Randomisation forces a brute search across ~28k ephemeral ports.

the change

// No web-facing API. Chromium's Windows TCP stack now
// uses the same randomised ephemeral port pool as macOS/Linux.
// Mitigation against off-path attacks (NAT rebinding, RST injection).
