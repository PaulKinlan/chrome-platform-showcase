The pre-v139 Windows allocator picked the lowest available port in a deterministic sweep — fast, but predictable. RFC 6056 (the random algorithm Chrome 139 turns on for Windows 2020 H1+) breaks that predictability with a 2-byte hash + offset, with a small allowance for backwards compatibility on older builds. Side-channel attacks that fingerprinted users by inducing rapid same-port reuse are no longer practical.
