Three Mediation Modes

conditional, silent, and the new immediate all return credentials without a top-level click — but they differ in whether they show UI, what they show, and how they fail. Click each, see the behaviour live.

conditional

await navigator.credentials.get({
  mediation: 'conditional',
  publicKey: { … },
});

UX: autofill chip inside the username field. Resolves only when the user picks the chip.

—

silent

await navigator.credentials.get({
  mediation: 'silent',
  publicKey: { … },
});

UX: no UI ever. Returns existing credential or null. Useful for “is the user still logged in?”.

—

immediate (139)

await navigator.credentials.get({
  mediation: 'immediate',
  publicKey: { … },
});

UX: shows the picker iff there's a known credential. If none, rejects fast with NotAllowedError.
