Manifest Configuration

The two files Chrome requires before migrating an installed PWA to a new origin: the new manifest with scope_extensions, and the old origin's .well-known association file.

Both sides must opt in. The new origin's manifest declares it accepts the old origin's install; the old origin's well-known file grants the new manifest permission to claim it. Chrome verifies both before migrating.

required files

New origin — manifest.json

https://app.example.com/manifest.json
{
  "name": "My App",
  "start_url": "/",
  "scope": "/",
  "scope_extensions": [
    {
      "origin":
        "https://old.example.com"
    }
  ],
  "related_applications": [
    {
      "platform": "webapp",
      "url": "https://old.example.com
              /manifest.json"
    }
  ]
}

Old origin — well-known

https://old.example.com/.well-known/web-app-origin-association
{
  "web_apps": [
    {
      "manifest":
        "https://app.example.com
         /manifest.json",
      "details": {
        "paths": ["/*"]
      }
    }
  ]
}

field reference

Field File Required Description
scope_extensions[].origin New manifest Yes Old origin to migrate installs from. Must be a valid origin string.
related_applications[].platform New manifest Yes Must be "webapp" for PWA-to-PWA migration.
related_applications[].url New manifest Yes Full URL of the old origin's manifest.json.
web_apps[].manifest Well-known Yes Full URL of the new origin's manifest.json — grants it permission to claim installs.
web_apps[].details.paths Well-known Recommended Path patterns on the old origin that the new app is allowed to handle. ["/*"] grants all paths.

migration scenarios

  • Subdomain to apex domain Old: app.example.com → New: example.com. Common when a product moves from a dedicated subdomain into the main site.
  • Brand rename Old: oldcompany.com → New: newcompany.com. Acquisitions and rebrands require a full domain migration.
  • HTTP to HTTPS Old: http://example.com → New: https://example.com. Retains installs when a site enables HTTPS for the first time (origins differ: scheme is part of the origin).
  • Port migration Old: localhost:3000 → New: localhost:5000. Useful for development workflows, but same mechanism applies.
