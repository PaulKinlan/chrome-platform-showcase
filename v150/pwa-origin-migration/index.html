PWA origin migration

Chrome 150 lets installed PWAs migrate to a new origin or scope without losing their install entry. A scope_extensions declaration in the web manifest declares the old origin as an associated scope, allowing Chrome to migrate the install automatically when the user visits the new origin.

concepts

  1. Migration Demo

    Inspects navigator.getInstalledRelatedApps() and displays the current install context, explaining what Chrome sees when a PWA has declared origin migration via the manifest.

  2. Manifest Configuration

    Reference for the manifest fields that enable origin migration: scope_extensions, related_applications, and the verification requirements that Chrome checks before performing the migration.

why it shipped

Organisations rebrand domains, migrate from subdomains to apex domains, or consolidate separate PWAs as their product evolves. Previously, any origin change meant a complete re-install — users lost their shortcuts, app data stored in origin-bound storage, and the PWA felt like a new app. Chrome 150 introduces a protocol for declaring origin equivalence: the new origin's manifest points back to the old, Chrome verifies both sides, and existing installs migrate automatically.

the change

// New origin's manifest.json — declares old origin as an extension
{
  "name": "My App",
  "short_name": "MyApp",
  "start_url": "https://app.example.com/",
  "scope": "https://app.example.com/",
  "scope_extensions": [
    {
      "origin": "https://old-subdomain.example.com"
    }
  ],
  "related_applications": [
    {
      "platform": "webapp",
      "url": "https://old-subdomain.example.com/manifest.json"
    }
  ]
}

// Old origin must also have a /.well-known/web-app-origin-association
// file granting association to the new origin:
// https://app.example.com/.well-known/web-app-origin-association
{
  "web_apps": [
    {
      "manifest": "https://app.example.com/manifest.json",
      "details": { "paths": ["/*"] }
    }
  ]
}
