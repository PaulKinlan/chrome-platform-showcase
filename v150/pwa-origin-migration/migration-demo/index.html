Reads your current install context and related installed apps to show what Chrome knows about this page as a PWA, and what fields are relevant to origin migration.

Origin migration requires both sides to declare the association: the new manifest includes scope_extensions, and the old origin hosts a /.well-known/web-app-origin-association file. Chrome performs this verification before migrating an existing install.

migration verification steps

Chrome performs these checks before migrating an installed PWA to the new origin:

  • 1 New origin's manifest.json includes scope_extensions listing the old origin
  • 2 New origin's manifest.json includes related_applications pointing to the old manifest URL
  • 3 Old origin hosts /.well-known/web-app-origin-association listing the new manifest URL
  • 4 Chrome fetches and verifies both the old manifest and the association file — both must be accessible
  • 5 On next visit to the new origin, Chrome presents the user with a migration prompt (or migrates silently per policy)

code

// Check for related installed apps
// (Chrome checks this during origin migration too)
if ('getInstalledRelatedApps' in navigator) {
  const apps = await navigator.getInstalledRelatedApps();

  if (apps.length) {
    // A related app is installed — migration may be in progress
    for (const app of apps) {
      console.log(app.platform, app.url, app.version);
    }
  } else {
    // No related apps — fresh install scenario
  }
}

// Detect if running as installed PWA
const isInstalled = window.matchMedia('(display-mode: standalone)').matches
  || window.matchMedia('(display-mode: minimal-ui)').matches
  || window.matchMedia('(display-mode: window-controls-overlay)').matches
  || navigator.standalone === true;
