Install State Simulator

Walk through a complete PWA origin migration from the user's perspective across four phases: pre-migration (app installed on old origin), developer deploys the migration, Chrome verifies and migrates the install, and post-migration state. Each phase shows what Chrome does, what the user sees, and what data survives.

The PWA is installed at https://app.old-company.com/app/. The user has a shortcut on their desktop/homescreen. App data lives in the old origin's storage partition.

Current state: app lives at https://app.old-company.com/app/
  • User
    App shortcut on device
    User installed the PWA from https://app.old-company.com/app/ at some point in the past. The app shortcut launches directly to this URL. All IndexedDB, localStorage, and service worker registrations are scoped to the old origin.
  • Chrome
    Install record in Chrome
    Chrome stores the manifest URL, app ID, and scope. The old manifest has no migration declarations yet. App ID at this point is typically the start URL.
    // Chrome's install record:
    { id: "https://app.old-company.com/app/", scope: "/app/" }

Data at this point

💾IndexedDB (old origin)present
🗄️localStorage (old origin)present
⚙️Service worker (old scope)active
🔔Push subscriptionsactive
The developer deploys changes to both origins. The new origin's manifest declares the old origin in scope_extensions. The old origin serves a web-app-origin-association file authorising the migration.
  • Server
    New origin: manifest.json updated
    Add scope_extensions pointing to old origin, related_applications with platform + URL + id, and set "id" to the new origin's start URL.
    // https://app.new-company.com/app/manifest.json
    { "id": "https://app.new-company.com/app/",
    "scope_extensions": [{"origin": "https://app.old-company.com"}],
    "related_applications": [{"platform": "webapp", "url": "https://app.old-company.com/app/", "id": "https://app.new-company.com/app/"}] }
  • Server
    Old origin: web-app-origin-association file deployed
    The old origin must serve a JSON file at /.well-known/web-app-origin-association authorising the new app. Without this, Chrome cannot verify both sides of the association.
    // GET https://app.old-company.com/.well-known/web-app-origin-association
    { "web_apps": [{"manifest_id": "https://app.new-company.com/app/",
    "details": {"paths": ["/app/*"]}}] }
On the user's next visit, Chrome fetches and verifies both files. If all checks pass, Chrome migrates the install record to the new origin. This is transparent to the user.
  • Chrome
    Fetch new manifest and verify scope_extensions
    Chrome fetches https://app.new-company.com/app/manifest.json and reads scope_extensions. Confirms the old origin is listed.
  • Chrome
    Fetch old origin's web-app-origin-association
    Chrome GETs https://app.old-company.com/.well-known/web-app-origin-association and verifies the manifest_id matches the new app's ID. This confirms the old origin consents to the migration.
  • Chrome
    Update install record
    Chrome updates the installed app's record from the old origin to the new one. The app shortcut continues to work — it now opens the new origin instead. The app ID is updated to the new app's declared ID.
    // Install record after migration:
    { id: "https://app.new-company.com/app/", scope: "/app/",
    origin: "https://app.new-company.com" }
Migration complete. The app shortcut now opens the new origin. The user sees no interruption. What happens to storage data depends on whether both origins share data before migration.

⚙️Service workerold SW still registered — may need update
🔔Push subscriptionsold origin — re-subscribe on new

📌App ID in Chromeupdated to new manifest_id
Storage note: origin-scoped APIs (IndexedDB, localStorage, Cache API) use origin as the partition key. Data on the old origin is not automatically copied to the new one. Your app's first load on the new origin should check for and migrate any critical user data from the old origin (using cross-origin fetch or redirects) before the old origin is retired.
