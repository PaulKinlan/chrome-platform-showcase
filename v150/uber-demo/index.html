Adaptive Music Player

Nine Chrome 150 features powering one music player. :playing/:paused/:muted drive all visual state in pure CSS. zoom animates album art in layout. alpha() tints overlays, light-dark() flips the background pattern, image() fills placeholders, focusgroup handles keyboard nav, flex-wrap: balance distributes the playlist, comma-separated container queries collapse both panels together, and text-fit sizes long track titles.

Chrome 150 features at work

feature role in this demo
:playing / :paused / :muted Drives every visual state — badge text, progress colour, strip visibility, mute button fill — zero JS for visuals
animatable zoom Album art expands with transition: zoom 0.4s ease when audio starts — a layout-affecting scale, not transform: scale()
alpha() Now-playing strip tint: alpha(var(--accent-blue), 8%) — one token, derived translucency, no extra variable
light-dark() + image values Topbar background uses light-dark(repeating-linear-gradient(…), repeating-linear-gradient(…)) to flip the bar-chart pattern per colour scheme
image() function Album art and track thumbnails use background-image: image(var(--accent-blue)) — a solid colour treated as an image
focusgroup Transport controls use focusgroup="toolbar" — a recognized behavior token that gives the horizontal toolbar ← → navigation. Tab enters and exits without JS roving tabindex.
flex-wrap: balance Playlist distributes 6 tracks into even rows — no orphaned single-item last row
comma-separated @container One rule — @container player (max-width: 460px), tracklist (max-width: 380px) — stacks both panels when either is narrow
text-fit: scale-down Long track titles scale their font size down to fit the available width — no overflow, no truncation, no viewport hacks

code

/* media element pseudo-classes (Chrome 150) — pure CSS state */
.player:has(audio:playing) .album-art { zoom: 1.06; }
.player:has(audio:muted)  .mute-btn  { background: var(--text-black); color: var(--bg-ivory); }
.player:has(audio:playing) .progress-fill { background: var(--accent-blue); }

/* animatable zoom (Chrome 150) */
.album-art { transition: zoom 0.4s ease; }

/* alpha() — translucent tint from one token (Chrome 150) */
.now-playing-strip { background: alpha(var(--accent-blue), 8%); }

/* light-dark() with image values (Chrome 150) */
.topbar {
  background-image: light-dark(
    repeating-linear-gradient(90deg, black 0px, black 1px, transparent 1px, transparent 10px),
    repeating-linear-gradient(90deg, white 0px, white 1px, transparent 1px, transparent 10px)
  );
}

/* image() function — color as background-image (Chrome 150) */
.album-art { background-image: image(var(--accent-blue)); }

/* focusgroup — toolbar arrow-key nav (Chrome 150) */
<div role="toolbar" class="transport" focusgroup="toolbar">...</div>

/* flex-wrap: balance — even row distribution (Chrome 150) */
.track-list { display: flex; flex-wrap: balance; }

/* comma-separated container queries — one rule, two containers (Chrome 150) */
@container player (max-width: 460px), tracklist (max-width: 380px) {
  .track-list { flex-direction: column; }
}

/* text-fit — scale font to fill container (Chrome 150) */
.track-title { text-fit: scale-down; white-space: nowrap; overflow: hidden; }
