display-state Media Feature

A live read-out of matchMedia("(display-state: …)") for all four values. A MediaQueryList change listener updates the table the instant the window's state flips — the mechanism a custom titlebar uses to swap its maximize and restore buttons in pure CSS.

The display-state media feature is not permission-gated, but it ships behind chrome://flags/#enable-desktop-pwas-additional-windowing-controls (or --enable-blink-features=DesktopPWAsAdditionalWindowingControls) in an installed desktop app. The recognition probe below reads the real parse result and can return false. The sibling display-mode: fullscreen query is shipping today, so the fullscreen button demonstrates the identical live-reaction pattern right now.

This mock window restyles via @media (display-state: maximized) and (display-mode: fullscreen). Trigger real fullscreen to watch the sibling pattern react live.

Fullscreen exercises the shipping display-mode media feature; the v150 display-state rows above light up once the flag is enabled in an installed app.

// Watch the window's display state in CSS — no permission needed:
@media (display-state: maximized) {
  #restore-button { display: flex; }
  #maximize-button { display: none; }
}

// …and in JS, reacting to every change:
const mql = window.matchMedia('(display-state: maximized)');
mql.addEventListener('change', (e) => {
  updateTitlebar(e.matches ? 'maximized' : 'restored');
});
console.log('maximized right now?', mql.matches);
