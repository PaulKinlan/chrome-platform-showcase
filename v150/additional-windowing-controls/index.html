v150 · Window Management

Additional Windowing Controls

Installed web apps can finally drive their own window chrome. Chrome 150 adds window.minimize(), window.maximize(), window.restore() and window.setResizable() — plus two CSS media features, display-state and resizable, so the UI can react to the window's state. Built for VDI clients and custom-titlebar apps that need to mirror a remote or bespoke window frame.

concepts

  1. Window State Controls

    The core JS surface: buttons that call await window.maximize(), minimize() and restore(). Each button invokes the real Promise-returning API, then reports whether it resolved or rejected — and why (missing permission, not an installed app, or unsupported).

  2. display-state Media Feature

    A live read-out of matchMedia("(display-state: …)") across all four values — normal, minimized, maximized, fullscreen. A MediaQueryList change listener updates the panel the moment the window state flips, exactly how a custom titlebar toggles its maximize/restore button.

  3. Resizable Lock

    Call await window.setResizable(false) to make a top-level window fixed-size, and watch the (resizable: true) media feature flip. Demonstrates the rule from the spec: user drag-resize is blocked, but Web APIs like resizeTo() still work.

  4. Custom Titlebar

    The VDI use case end-to-end: a bespoke titlebar drawn in HTML with app-region: drag, non-draggable window-control buttons wired to the AWC APIs, and CSS @media (display-state: maximized) rules that swap the maximize and restore buttons automatically.

why it shipped

Virtual Desktop Infrastructure (VDI) clients — Citrix, Omnissa and similar — present remote application windows inside a local browser window. Until now the web platform gave them no way to relay a user's intent when they clicked a remote or custom minimize/maximize control: the local window and the remote window drifted out of sync, and titlebar buttons were inoperative and confusing. Additional Windowing Controls closes that gap. Installed apps can request minimize/maximize/restore and lock resizing (all gated by the window-management permission), and read the resulting state back through the display-state and resizable media features (which are not permission-gated, so CSS can react for free). The same surface unlocks compelling custom-titlebar experiences for any installed web app.
