Window State Controls

Call the real Promise-returning window-state methods and see exactly what they return. Each button invokes await window.maximize(), minimize() or restore() and reports whether the platform resolved it, rejected it, or never exposed the method at all.

To run this for real, these APIs need an installed desktop PWA with the window-management permission, behind a flag today:
  1. Open chrome://flags/#enable-desktop-pwas-additional-windowing-controls → Enabled, or launch Chrome with --enable-blink-features=DesktopPWAsAdditionalWindowingControls.
  2. Install this site as an app (the methods only exist in an installed app window, not a browser tab).
  3. Grant the window-management permission when prompted.
If any method is missing below, the demo says so honestly and never fakes a result.

try {
  await window.maximize();   // resolves once the window is maximized
} catch (err) {
  // rejected: no permission, not an installed app, or state unreachable
  console.log('maximize() rejected:', err.name, err.message);
}
