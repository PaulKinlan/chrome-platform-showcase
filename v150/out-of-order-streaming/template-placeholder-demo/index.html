Template Placeholder Demo

Four named <template for="…"> slots are painted immediately. Content streams in out of order — fast sections fill first, slow sections arrive later. No JavaScript moves any content; the browser's HTML parser does it all.

Chrome 150 required for native <template for> + processing instruction support. This demo simulates the declarative mechanism using JavaScript to illustrate the concept in any browser. The server-side native path requires no JS at all.

server-side code (Chrome 150+)

<!-- Server emits placeholders immediately -->
<template for="nav"><div class="skeleton">…</div></template>
<template for="hero"><div class="skeleton">…</div></template>
<template for="feed"><div class="skeleton">…</div></template>
<template for="recs"><div class="skeleton">…</div></template>

<!-- ... rest of page ... -->

<!-- Server flushes chunks as they're ready (any order) -->
<?start nav?>  <nav>…</nav>   <?end nav?>    <!-- instant -->
<?start hero?> <h1>…</h1>    <?end hero?>   <!-- 300 ms -->
<?start feed?> <ul>…</ul>    <?end feed?>   <!-- 900 ms -->
<?start recs?> <div>…</div>  <?end recs?>   <!-- 1.8 s -->
