Out of order streaming

Chrome 150 introduces <template for="…"> and processing instruction ranges (<?start name?> / <?end name?>) — a declarative server-side mechanism that lets HTML arrive and be inserted into placeholder positions in the document out of order, without any client-side JavaScript.

concepts

  1. Template Placeholder Demo

    Watch a simulated server stream HTML chunks out of order into named <template for="…"> placeholders. The page layout is painted immediately with skeleton slots; content fills in as each chunk arrives — no layout shift, no waiting for slow sections.

  2. Streaming Comparison

    Side-by-side: traditional top-to-bottom streaming where a slow section blocks everything below it, versus out-of-order streaming where fast sections appear immediately into their placeholders while slow sections catch up independently.

why it shipped

Modern server-side rendering often produces some sections of a page quickly (navigation, above-the-fold content, cached data) while other sections are slow (personalised recommendations, database queries, AI-generated content). With classic in-order streaming, any slow section blocks the sections below it from being parsed. React's <Suspense> and similar framework mechanisms solve this by streaming out of order with client-side JavaScript rehydration — but that requires a large JS runtime and introduces a "hydration cliff". Chrome 150's <template for> mechanism moves the out-of-order patching into the HTML parser itself: the server emits a placeholder early and streams the content later, anywhere in the byte stream, and the browser slots it into place without any JavaScript.

the mechanism

<!-- 1. Emit a named placeholder early in the page -->
<template for="product-recs">
  <div class="skeleton">Loading recommendations…</div>
</template>

<!-- 2. Rest of the page renders without waiting -->
<main> … fast content … </main>

<!-- 3. Later in the stream (even after </body>), send the content -->
<?start product-recs?>
  <section><h2>Recommended for you</h2> … </section>
<?end product-recs?>

<!-- Browser replaces the <template> with the streamed content -->
<!-- No JavaScript required -->
