Out-of-order streaming

Chrome 150 enables streamed HTML responses to patch existing document ranges as soon as each patch chunk arrives. A shell marks ranges with processing instructions, and later <template for> chunks update those ranges without imperative JavaScript.

  6. AI Chat Simulation

    AI token streaming is the primary real-world use case. Configure a slow "thinking" chunk anywhere in the token sequence, then watch old in-order mode hold back all subsequent tokens while Chrome 150 displays each one immediately. Side-by-side chat panels, per-token dispatch timeline, and time-to-first-token metrics show exactly how much perceived latency the change removes.

why it shipped

HTML streaming traditionally updates the document in DOM order. If a server cannot produce an early hero, carousel, or result section yet, unrelated sections that are ready still have to wait behind that byte sequence. Declarative partial updates let the response ship a useful shell first, then interleave later <template for> patches for named ranges anywhere in the document.

the change

<main>
  <section>
    <?start name="hero">Loading hero...<?end>
  </section>
  <aside>
    <?start name="related">Loading related links...<?end>
  </aside>
</main>

<template for="related">
  <nav>Related links that were ready first</nav>
</template>

<template for="hero">
  <h2>Hero content that finished later</h2>
</template>
