AI chat simulation

AI chat responses are the canonical use case for out-of-order streaming: tokens arrive as the model generates them, and every millisecond of head-of-line blocking visibly delays the text appearing. This demo simulates one slow "thinking" chunk mid-stream and shows how old in-order mode holds back all subsequent tokens while Chrome 150 dispatches each one immediately.

In a real AI API (fetch + ReadableStream), the server pushes tokens as SSE or raw bytes. A slow token at position N no longer blocks tokens N+1…end from being dispatched to your reader.read() call. The cursor moves sooner; the user reads sooner.

// AI streaming response with ReadableStream
async function streamAIResponse(prompt, onToken) {
  const resp = await fetch('/api/ai/generate', {
    method: 'POST',
    body: JSON.stringify({ prompt }),
    headers: { 'Content-Type': 'application/json' },
  });

  // Chrome 150: out-of-order delivery — tokens arrive as they're generated,
  // even if the server sends them at different rates
  const reader = resp.body.getReader();
  const decoder = new TextDecoder();

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    // Each chunk is dispatched immediately in Chrome 150
    // In old browsers, a slow chunk N holds up chunks N+1, N+2, ...
    const token = decoder.decode(value, { stream: true });
    onToken(token);  // append to chat bubble as it arrives
  }
}
