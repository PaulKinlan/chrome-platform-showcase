      what changes

      DOM-order streaming

      The parser receives bytes in sequence. If the server cannot produce an early part of the document, later bytes that are ready still wait behind it before they can affect the page.

      Declarative partial updates

      The shell declares named ranges with processing instructions. Later chunks carry <template for="name"> patches that can land wherever their target range is, even when other ranges are still loading.

      markup contract

      <section>
  <?start name="hero">Loading hero...<?end>
</section>

<section>
  <?start name="results">Loading results...<?end>
</section>

<template for="results">
  <article>First result</article>
  <?marker name="results">
</template>

<template for="hero">
  <h2>Hero content finally ready</h2>
</template>

      edge cases to watch

      • Templates with a valid for target apply declaratively instead of becoming visible content.
      • A missing or invalid target should remain inspectable in the DOM as an unapplied template.
      • Repeated patches can use a <?marker> to keep appending into the same range.
      • Nested ranges must close in stack order because <?end> closes the nearest unmatched <?start>.
