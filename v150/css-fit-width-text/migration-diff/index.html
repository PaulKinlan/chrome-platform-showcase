Four common patterns — viewport-unit sizing, clamp(), JavaScript resize loops, and SVG textLength — compared against the native text-fit equivalent. Toggle between old and new in a live preview, and read why the native approach is simpler and more correct.

compatibility summary

Old technique Limitation text-fit advantage
font-size: 5vw Only accurate at one specific container/viewport ratio. Wrong for nested containers, sidebars, responsive grids. text-fit measures the actual container at render time.
clamp(1rem, 4vw, 3rem) Approximation — text rarely fills width exactly. Requires hand-tuning per text string and container size. text-fit: balance fills exactly — no tuning needed.
JS resize loop (measureText) Runs after paint, causes layout shift. Breaks SSR, complex in frameworks, not CSS-animatable. text-fit runs during layout — no shift, no script.
SVG textLength attribute Stretches letter spacing, not font size. Looks wrong for large text. Only works in SVG context. text-fit scales font-size — glyphs look natural.
transform: scaleX() Stretches glyphs horizontally — distorted appearance. Not accessible. text-fit scales uniformly — no distortion.
