Streaming Use Case

Processing instructions with the reserved targets start and end power the out-of-order streaming mechanism. The browser uses them as live range anchors to replace a <template for="…"> placeholder with streamed content — no JavaScript required.

Chrome 150 required for native out-of-order streaming via processing instructions. This page explains the mechanism — the actual streaming works at the HTML parser level and cannot be fully emulated in a static demo page.

how it works, step by step

  1. 1
    Server emits a named placeholder early in the response. A <template for="slot-name"> element is placed where the content should appear. The server may include a skeleton inside it.
    <template for="recommendations"> <div class="skeleton">Loading…</div> </template>
  2. 2
    The rest of the page streams normally. Content below the placeholder — including the footer — arrives and renders without waiting for the slow section to complete.
  3. 3
    When the slow content is ready, the server appends PI-bounded HTML. The <?start name?> and <?end name?> processing instructions act as a range marker around the replacement content, anywhere later in the byte stream.
    <?start recommendations?> <section> <h2>Recommended for you</h2> <ul>…</ul> </section> <?end recommendations?>
  4. 4
    The browser replaces the placeholder, no JavaScript needed. When the parser encounters <?start recommendations?>, it finds the matching <template for="recommendations"> in the live DOM, removes it, and inserts the streamed content in its place. The PI nodes themselves are consumed and do not appear in the final DOM.

The start and end target names are reserved by the browser for the streaming mechanism. All other targets produce ProcessingInstruction nodes in the live DOM.

complete server-side example

<!DOCTYPE html>
<html>
<head><title>My page</title></head>
<body>

  <!-- Placeholder with skeleton — emitted immediately -->
  <template for="recs">
    <div class="skeleton">Loading recommendations…</div>
  </template>

  <!-- Rest of page renders right away -->
  <main>
    <h1>Welcome</h1>
    <p>Fast content here.</p>
  </main>
  <footer>© 2025</footer>

  <!-- Server flushes this ~1 second later, after DB query -->
  <?start recs?>
    <section class="recommendations">
      <h2>Recommended for you</h2>
      <ul><li>Item 1</li><li>Item 2</li></ul>
    </section>
  <?end recs?>

</body>
</html>
