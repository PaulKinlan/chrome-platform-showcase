Constraint Configurator

The <usermedia> element passes constraints to the underlying getUserMedia() call — video resolution, frame rate, facing mode, audio echo cancellation, noise suppression. Build your constraint set with the controls, watch the generated markup and getUserMedia() equivalent update live, then click "Test constraints" to run them and see what the browser actually negotiates.

<!-- usermedia element: constraints as attributes -->
<usermedia
  video
  audio
  data-constraints='{"video":{"width":{"ideal":1280},"facingMode":"user"},"audio":{"echoCancellation":true}}'
>
  <button>Start camera</button>
</usermedia>

<!-- The element fires "activate" when the user clicks and access is granted -->
<script>
  document.querySelector('usermedia').addEventListener('activate', e => {
    document.querySelector('video').srcObject = e.stream;
  });
</script>
