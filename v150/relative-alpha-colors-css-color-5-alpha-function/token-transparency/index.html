Token Transparency

Four design tokens — --brand, --success, --danger, --neutral — generate their own transparent variants on demand using alpha(). No extra variables, no color pre-computation.

--brand: #0022ff
var(--brand) — 100%
alpha(var(--brand), 60%)
alpha(var(--brand), 30%)
alpha(var(--brand), 10%)
--success: #0a8f5c
var(--success)
alpha(var(--success), 60%)
alpha(var(--success), 30%)
alpha(var(--success), 10%)
--danger: #d11a2a
var(--danger)

var(--neutral)

In practice — alert banners from one token each

ℹ️ Info: background is alpha(var(--brand), 10%), border is var(--brand).
✅ Success: background is alpha(var(--success), 10%).
⚠️ Warning: background is alpha(var(--danger), 10%).
/* One token, many transparencies — Chrome 150 */
:root { --brand: #0022ff; }

.btn-ghost    { background: alpha(var(--brand), 10%); }
.btn-subtle   { background: alpha(var(--brand), 30%); }
.alert-bg     { background: alpha(var(--brand), 15%); }
.overlay      { background: alpha(var(--brand), 60%); }
