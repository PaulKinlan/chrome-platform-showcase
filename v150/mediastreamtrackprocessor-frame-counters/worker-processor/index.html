Worker Processor

Move the MediaStreamTrackProcessor into a dedicated Worker thread — the recommended architecture for heavy per-frame transforms — and read totalFrames / discardedFrames from the worker back to the main thread via postMessage. The main thread only receives the health stats, never the raw frames.

This demo uses a canvas-based synthetic MediaStream (no camera permission required). The processor lives in an inline Blob Worker. Counter stats are sent to main thread at 1 Hz. Slow down the consumer with the slider to induce frame drops.

// main.js
const track = canvas.captureStream(30).getVideoTracks()[0];
const worker = new Worker(workerUrl);
worker.postMessage({ type: 'start', track }, [track]);

worker.onmessage = ({ data }) => {
  if (data.type === 'stats') {
    console.log(data.totalFrames, data.discardedFrames);
  }
};

// worker.js
self.onmessage = ({ data }) => {
  if (data.type !== 'start') return;
  const processor = new MediaStreamTrackProcessor({ track: data.track });
  const reader = processor.readable.getReader();

  setInterval(() => {
    self.postMessage({
      type: 'stats',
      totalFrames: processor.totalFrames,
      discardedFrames: processor.discardedFrames,
    });
  }, 1000);

  (async () => {
    while (true) {
      const { done, value: frame } = await reader.read();
      if (done) break;
      // ... per-frame work ...
      frame.close();
    }
  })();
};
