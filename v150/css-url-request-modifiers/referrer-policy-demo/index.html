Referrer Policy Demo

The referrer-policy() modifier inside url() controls what referrer information is included in the HTTP request for a CSS-referenced resource — the same policies available on <img referrerpolicy> and the Referrer-Policy HTTP header, now directly in CSS.

Without referrer-policy(), CSS resource requests use the page's document referrer policy (set by the Referrer-Policy HTTP header or <meta> tag). The modifier lets individual CSS resource references override that policy.

policy values

Policy value Cross-origin request sends Same-origin request sends Notes
no-referrer nothing nothing Maximum privacy
no-referrer-when-downgrade full URL (HTTPS→HTTPS) full URL Nothing on HTTPS→HTTP downgrade
strict-origin-when-cross-origin origin only full URL Default for most browsers
origin origin only origin only Never sends path or query
same-origin nothing full URL Cross-origin: no referrer
strict-origin origin (HTTPS→HTTPS only) origin only No referrer on downgrade
origin-when-cross-origin origin only full URL Common privacy-conscious choice
unsafe-url full URL always full URL always Leaks URLs — avoid

why this matters for CSS resources

Privacy leak scenario

Imagine a page at https://myapp.com/user/profile?id=1234 that uses a CDN image as a background. Without a referrer policy, the CDN receives the full URL in the Referer header — including the user's ID. The CDN operator can log user IDs from referrer headers.

With referrer-policy(strict-origin) on the url(), the CDN only sees the origin (https://myapp.com), not the path or query string.

code

/* Default: uses document's referrer policy */
.hero {
  background-image: url("https://cdn.example/hero.png");
}

/* Override for this specific resource */
.hero {
  background-image: url("https://cdn.example/hero.png"
    referrer-policy(no-referrer));
}

/* Practical pattern: strict for CDN images to avoid leaking user paths */
.user-avatar {
  background-image: url("https://cdn.example/avatar.png"
    referrer-policy(strict-origin));
}

/* Combined with cross-origin for CORS resources */
@font-face {
  font-family: 'PrivateFont';
  src: url("https://fonts.corp.example/font.woff2"
    cross-origin(use-credentials)
    referrer-policy(same-origin));
}

/* HTML element equivalent (for reference):
   <img src="https://cdn.example/img.png"
        referrerpolicy="strict-origin"> */
