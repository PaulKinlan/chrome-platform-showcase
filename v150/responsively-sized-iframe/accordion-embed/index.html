Accordion Embed

A simulated FAQ accordion embedded as an iframe. The old approach clips or scrolls because the height is fixed. With allow-responsive-sizing + Supports-Responsive-Sizing: 1, the iframe grows and shrinks with the accordion — zero JavaScript on either side.

Click questions in either panel to expand / collapse them and watch how each approach handles the height change.

What you're seeing: Both panels contain identical accordion markup. Left panel: the iframe is fixed at 180px, so content is clipped when items expand. Right panel: the iframe height tracks the content exactly, as the browser propagates the embedded document's layout overflow — no ResizeObserver, no postMessage, no script on either side.

Old — parent (embedder)

<!-- Fixed height — must guess -->
<iframe
  src="/faq-embed"
  style="height:180px; width:100%"
  id="faqFrame"
></iframe>

<script>
// Listen for child's size message
window.addEventListener('message', e => {
  if (e.data.type !== 'resize') return;
  document.getElementById('faqFrame')
    .style.height = e.data.height + 'px';
});
</script>

New — parent (embedder)

<!-- Just add the attribute -->
<iframe
  src="/faq-embed"
  allow-responsive-sizing
></iframe>

<!-- No script needed. The browser
     reads Supports-Responsive-Sizing: 1
     from the embedded document's
     response headers and handles
     propagation automatically. -->

Old — child (embedded doc)

// In the embedded FAQ page
const ro = new ResizeObserver(entries => {
  const h = document.documentElement
    .scrollHeight;
  parent.postMessage(
    { type: 'resize', height: h },
    '*'   // wide-open origin — risky
  );
});
ro.observe(document.documentElement);

New — child (embedded doc)

# HTTP response header on /faq-embed:
Supports-Responsive-Sizing: 1

<!-- The page itself needs zero
     JavaScript. No ResizeObserver,
     no postMessage, no parent origin
     check. The browser observes the
     document's layout overflow and
     pushes the dimension change to
     the iframe box in the parent. -->
