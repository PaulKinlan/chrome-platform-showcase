postMessage Comparison

Both panels show a simulated embedded widget that grows as you add items. The left uses the old postMessage workaround to relay scrollHeight to the parent. The right shows what the new declarative allow-responsive-sizing + header approach replaces it with — zero JavaScript required on either side.

The new responsive sizing feature requires the server to send a Supports-Responsive-Sizing: 1 header on the embedded document. Because this page is static HTML, the left panel uses a working same-origin postMessage simulation and the right panel shows the declarative code that would replace it.

Child page (old postMessage approach)
// embedded.html
function notifySize() {
  parent.postMessage({
    type: 'resize',
    height: document.body.scrollHeight
  }, '*');
}
const obs = new ResizeObserver(notifySize);
obs.observe(document.body);
notifySize();
Parent page (old postMessage approach)
// parent.html
window.addEventListener('message', e => {
  if (e.data.type === 'resize') {
    iframe.style.height = e.data.height + 'px';
  }
});
Child page (Chrome 150 declarative)
/* No JavaScript needed */

/* HTTP response header on embedded.html: */
Supports-Responsive-Sizing: 1
Parent page (Chrome 150 declarative)
<!-- No JavaScript needed -->
<iframe
  src="embedded.html"
  allow-responsive-sizing>
</iframe>
