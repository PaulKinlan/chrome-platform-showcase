Chrome 150 makes anchor-name matching case-sensitive in quirks mode, aligning it with standards mode. Explore the before/after behaviour and test name pairs live.

Checking CSS Anchor Positioning support…

Three scenarios

Standard mode (always)

Documents with <!DOCTYPE html> have always used case-sensitive matching. --myAnchor never matched --MYANCHOR.

Unchanged by Chrome 150.

Quirks mode — pre-150

Documents without a doctype ran in quirks mode. Anchor name matching was case-insensitive: --myAnchor matched --MYANCHOR.

This was a spec deviation that caused portability bugs.

Quirks mode — Chrome 150+

Chrome 150 aligns quirks mode with standards: matching is now case-sensitive in both modes.

--myAnchor and --MYANCHOR are now different identifiers everywhere.
