Focus Hijack Guard

Two embedded frames each try to steal keyboard focus on a timer — simulating a widget that calls element.focus() without any user interaction. The right frame carries allow="focus-without-user-activation 'none'". Watch the status badges update every 2.5 s.

Both iframes load the same page (frame.html) which calls input.focus() every 2.5 seconds with no user gesture. The left frame succeeds every time. The right frame's focus calls are silently dropped — the policy is enforced transparently, with no JavaScript error thrown.

the markup

<!-- No restriction: focus() inside succeeds --> <iframe src="embed.html"></iframe> <!-- Restricted: focus() without activation silently blocked --> <iframe src="embed.html" allow="focus-without-user-activation 'none'"> </iframe> <!-- HTTP header equivalent (covers the whole page tree): --> Permissions-Policy: focus-without-user-activation=()

what the policy covers

  • element.focus() — silently no-ops without user activation
  • window.focus() — same treatment
  • autofocus HTML attribute — ignored on load
  • dialog.showModal() — focus on the dialog is suppressed
  • Popover focusing via showPopover()

User-initiated focus (tab key, mouse click, touch) is never affected.
