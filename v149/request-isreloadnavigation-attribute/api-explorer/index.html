API Explorer

Construct Request objects and read isReloadNavigation. The attribute is always false on manually constructed requests — it is only true inside a service worker's fetch event when the browser initiates a reload navigation.

Chrome 149 required. The attribute exists in Chrome 149+. On older browsers, isReloadNavigation returns undefined. The real value of the flag is most meaningful inside a service worker's fetch event — see the Service Worker Demo.

attribute reference

Property Type Context where meaningful Chrome
Request.isReloadNavigation boolean Service worker fetch event — true when user reloaded 149+
Request.mode string All contexts — e.g. "navigate", "cors", "no-cors" All
Request.destination string All contexts — e.g. "document", "script", "image" All
Request.cache string All contexts — "no-cache" on hard reload All

service worker pattern

// Inside service worker fetch handler
self.addEventListener('fetch', event => {
  const req = event.request;

  // Only applies to top-level navigation requests
  if (req.mode === 'navigate' && req.isReloadNavigation) {
    // User reloaded — bypass cache for fresh HTML
    event.respondWith(
      fetch(req).catch(() => caches.match(req))
    );
    return;
  }

  // Normal navigation — serve cached shell for speed
  event.respondWith(
    caches.match(req).then(cached => cached || fetch(req))
  );
});
