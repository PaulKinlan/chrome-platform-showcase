Shape Comparator

Four shapes — teardrop, arc, wave, chevron. Left column: polygon() approximation (Chrome 37+). Right column: path() with exact curves (Chrome 149+). The text flow reveals every difference.

Note: In Chrome 149+, both columns use native functions. In older browsers, the right column's path() value is ignored and the float loads without a shape — text won't wrap at all on the right, which itself demonstrates why the feature matters.

polygon() — Chrome 37+ (faceted)

The polygon approximation uses straight line segments. At low point counts, faceting is obvious on any curved shape. Increasing points helps but never achieves true curvature — and each added point makes the CSS harder to write and maintain.

path() — Chrome 149+ (smooth curves)

The path() function uses SVG path syntax: cubic Bézier curves (C), quadratic curves (Q), arcs (A). Curves are mathematically perfect at any scale. The CSS is shorter and exactly matches the visual clip-path shape — one source of truth.
