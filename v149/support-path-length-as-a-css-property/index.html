path-length CSS property

Chrome 149 makes path-length a presentational CSS property for SVG paths. Previously, overriding the SVG pathLength attribute required touching the SVG markup. Setting path-length in CSS enables stylesheet-based control of stroke-dash animations, percentages in stroke-dasharray, and path-relative measurements.

concepts

  1. Stroke Animation Demo

    Animate SVG stroke-dashoffset to draw paths. With CSS path-length, you set the normalized length (e.g. 100) in a stylesheet and compute dasharray offsets against that fixed unit — no measuring path lengths at runtime.

  2. Path Length Override

    Compare a path with its native length vs an overridden path-length. Shows how different path-length values scale the effective stroke-dasharray units, useful for creating repeating patterns at a consistent visual density.

why it shipped

SVG's pathLength attribute lets authors declare a virtual path length, making stroke-dasharray values relative to that unit instead of the true geometric length. The canonical draw-on-scroll and loader animations use this: set pathLength="1", then animate stroke-dashoffset from 1 to 0. Before Chrome 149, this required the pathLength attribute on each SVG element — the attribute couldn't be set in CSS. Chrome 149 makes path-length a presentation attribute accessible from CSS, enabling stylesheet-driven control with custom properties, @keyframes, and media queries.

the CSS property

/* SVG attribute (pre-Chrome 149) */
<path d="M10 80 Q95 10 180 80" pathLength="1" />

/* CSS property (Chrome 149+) */
path {
  path-length: 1;         /* set from CSS — no attribute needed */
  stroke-dasharray: 1;    /* full dash */
  stroke-dashoffset: 1;   /* start fully hidden */
  animation: draw 2s forwards;
}

@keyframes draw {
  to { stroke-dashoffset: 0; } /* reveal full path */
}

/* With custom properties — easy to parameterize */
.my-path {
  --total: 100;
  path-length: var(--total);
  stroke-dasharray: var(--total);
}
