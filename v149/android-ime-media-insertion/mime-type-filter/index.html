MIME Type Filter

When an Android IME inserts media, the beforeinput event fires with inputType === "insertFromPaste" and a dataTransfer carrying the MIME type. Toggle which types your editor accepts and use the simulate buttons to replay the decision — the filter calls event.preventDefault() on rejected types.

The filter logic
// Accepted types list — built from checkboxes above const acceptedTypes = new Set(['image/gif', 'image/png', 'image/jpeg']); const allowImageWildcard = wildcardToggle.checked; const allowEmptyType = emptyTypeToggle.checked; const denySvg = svgDenyToggle.checked; editor.addEventListener('beforeinput', (event) => { if (event.inputType !== 'insertFromPaste') return; const dt = event.dataTransfer; if (!dt) return; // Check each file's MIME type for (const file of dt.files) { const type = file.type || ''; const allowed = acceptedTypes.has(type) || (allowImageWildcard && type.startsWith('image/')) || (allowEmptyType && type === ''); if (!allowed || (denySvg && type === 'image/svg+xml')) { event.preventDefault(); // Block the insertion console.warn('Blocked MIME type:', type || '(empty)'); return; } } // All types accepted — allow the default insertion to proceed });
