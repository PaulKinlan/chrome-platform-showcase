common use cases for autocorrect="off"

  • Username / email fields — corrections mangle usernames like j_smith99
  • Password fields — already handled by type="password", but explicit autocorrect="off" adds a second layer
  • Code editors — keywords, identifiers, and symbols must not be altered
  • URL / slug inputs — corrections change paths and break links
  • Technical / medical terminology — domain-specific terms that autocorrect "fixes" incorrectly
