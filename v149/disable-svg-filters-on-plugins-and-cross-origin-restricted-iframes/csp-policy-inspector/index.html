Key takeaway: The new restriction targets only filter attributes that reference SVG <filter> elements. The CSS filter property (blur, brightness, contrast, etc.) on iframes is unaffected. Use the matrix above to identify which frame configurations in your app need remediation.
<!-- SVG filter applied to a cross-origin iframe → BLOCKED in Chrome 149 --> <svg> <filter id="blur"><feGaussianBlur stdDeviation="4"/></filter> <foreignObject> <iframe src="https://cross-origin.example" filter="url(#blur)"></iframe> </foreignObject> </svg> <!-- CSS filter on cross-origin iframe → still allowed --> <iframe src="https://cross-origin.example" style="filter: blur(4px)"></iframe>
