Combo Detector

Fighting-game style combos require catching every button press in sequence, including rapid inputs between animation frames. Polling misses sub-frame presses entirely — but rawgamepadinputchange events fire for every physical button transition, making multi-input combos reliable for the first time on the web.

Why polling misses combos: a "double-tap right" combo requires detecting two → presses within ~200ms, which may both land in the same animation frame. Poll-based code sees only one sample per rAF tick — the second → is invisible. rawgamepadinputchange fires for each physical transition, so both presses are delivered even if they are sub-frame.
