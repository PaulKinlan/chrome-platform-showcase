Cold vs Warm

Each run loads an iframe containing a large inline script. The first time, the bytecode cache is cold; on subsequent runs in v149+, it should warm up and the parse/compile slice should shrink.

runload (ms)script compile-marknote

what's being measured

The iframe's HTML contains a single <script> block ~80KB long. The page measures the time between iframe load firing and the script's last statement executing — a rough proxy for "how long did the script-block path take?" In v149+, the second and subsequent runs reuse cached bytecode and the script-block time should drop.
