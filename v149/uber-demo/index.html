Chapter I The Origin of Shapes

Before CSS could draw arbitrary shapes, designers were limited to rectangles and a fixed set of polygon primitives. Text wrapping required exact pixel measurements calculated in JavaScript. Every responsive breakpoint broke the layout. The web needed a path language that worked with CSS units.

Chapter II Rules in the Gaps

Adding visual separators between grid cells required adding extra markup, fighting doubled borders with :first-child workarounds, or accepting the layout cost of real borders. The gap was a dead zone. column-rule and row-rule reclaim it.

Chapter III Scrolls that Resolve

For years developers polled scrollTop in a loop or guessed a setTimeout duration to detect when smooth scrolling had finished. Now scrollIntoView() returns a Promise. The Next Chapter button below chains using await — no timers, no polling, no guessing.

Chapter IV Crisp Pixels & Sharp Paths

Pixel art scaled in CSS blurred before image-rendering: crisp-edges was available. Meanwhile, SVG stroke animations required JavaScript to compute getTotalLength(). Now path-length: 100 lets you write stroke-dashoffset: calc(100 - var(--pct)) — pure CSS, no geometry.

Chrome 149 features at work

shape()
Each card is clipped to a distinct corner-cut polygon via clip-path: shape(…), and the pixel-art float wraps text via shape-outside: shape(…) — all in percentage units, fully responsive.
column-rule / row-rule
column-rule and row-rule draw the solid and dashed separator lines inside the grid gaps. Zero extra markup, zero layout impact.
scroll promise
Next Chapter calls await el.scrollIntoView({ behavior: 'smooth' }) and reports the result (interrupted: false/true) once the scroll settles. No setTimeout required.
image-rendering: crisp-edges
Each 8×8 pixel-art canvas is scaled to 56px via CSS. image-rendering: crisp-edges / pixelated prevents the bilinear blur that would otherwise soften the pixels.
Intl.Locale.prototype.variants
The locale picker calls new Intl.Locale(tag).variants (Chrome 149 API) to read variant subtags as an array — de-1901 → ["1901"], sl-rozaj-biske → ["rozaj","biske"].
path-length
The progress ring circle sets path-length: 100 in CSS. The animation is just stroke-dashoffset: calc(100 - var(--progress)) — no getTotalLength(), no JavaScript geometry.
