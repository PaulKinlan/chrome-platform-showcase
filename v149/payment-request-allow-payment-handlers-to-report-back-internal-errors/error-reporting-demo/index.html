Error Reporting Demo

Simulates various payment handler error responses and shows how the merchant page receives the structured error detail from each scenario in Chrome 149+.

Simulation only. The Payment Request API requires user activation and a secure context with a registered payment method. This demo simulates the error-reporting flow using the same data shapes — real integration is identical in structure but goes through PaymentRequestEvent.respondWith() in a service worker.

trigger a handler error

Account balance too low to cover the transaction amount

3ds_failed

3D Secure authentication challenge was not completed

network_error

Handler could not reach the payment network

success

Payment succeeds — no error detail

merchant-side code

try {
  const response = await request.show();
  await response.complete('success');
  showSuccessUI();
} catch (err) {
  // Chrome 149+: err.detail contains the handler's structured error
  const detail = err.detail;

  if (detail?.error === 'card_expired') {
    redirectToCardUpdate(detail.message);
  } else if (detail?.error === 'insufficient_funds') {
    showFundsError(detail.message);
  } else if (detail?.error === '3ds_failed') {
    offerRetry(detail.message);
  } else {
    showGenericError(err.message);
  }
}
