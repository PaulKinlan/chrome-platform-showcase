text-overflow: ellipsis on focus

When a user focuses or edits an element with text-overflow: ellipsis, Chrome 149 automatically switches to clip — revealing the full content for interaction. No JavaScript required.

concepts

  1. Focus Reveal

    Click any of the truncated cells — they reveal their full text on focus. Tab through them. The ellipsis returns when you leave. All driven by the browser's built-in behavior, no JavaScript.

  2. Editable Comparison

    Side-by-side: a table with contenteditable cells vs. one without. Click a truncated cell — the editable one reveals on focus in Chrome 149; the read-only one stays clipped.

why it shipped

Before Chrome 149, text-overflow: ellipsis on a focused or editable element was a UX trap: the user would click to edit a text field, the cursor would appear inside clipped, invisible text, and there was no way to navigate to the hidden content without manually scrolling. Form inputs (<input>, <textarea>) already handled this correctly, but contenteditable elements and other scrollable text containers did not. Chrome 149 aligns all text containers: on focus or caret entry, ellipsis gives way to clip so the full content is visible and navigable. The ellipsis resumes when focus leaves.
