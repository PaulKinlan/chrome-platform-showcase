A single-producer / single-consumer ring buffer in SharedArrayBuffer. Two workers swap messages as fast as they can. Toggle Atomics.pause() in the consumer's spin and watch the messages-per-second figure converge to fewer wasted cycles — without dropping throughput when the producer is busy.

why pause helps a lock-free queue

A SPSC ring buffer is the canonical zero-allocation hand-off between two threads. The consumer typically spins on a head/tail counter; without a pause hint, every probe burns power and competes with the producer for the same cacheline. Atomics.pause(n) on each empty probe lets the CPU yield SMT siblings, lower the core clock, and reduce memory bus traffic — the producer's writes land sooner, and the overall throughput stays the same or improves while energy drops.

// consumer, hot loop
while (true) {
  const v = Atomics.load(buf, headIdx);
  if (v !== EMPTY) { handle(v); Atomics.store(buf, headIdx, EMPTY); break; }
  Atomics.pause();   // hint — no scheduler yield, just “wait a moment”
}
