Connection state-machine debugger

Drive a synthetic serial / RFCOMM port through every transition: granted → connected → reading → physically unplugged → reconnect. The state diagram highlights the current node, the event ticker records every connect / disconnect firing with the value of port.connected at the time, and the right-hand panel shows what a Web Serial app would observe via the Chrome 130 API surface.

the code


const ports = await navigator.serial.getPorts();
for (const port of ports) {
  console.log("connected (Chrome 130):", port.connected);

  // RFCOMM ports now also fire 'connect' and 'disconnect' events
  port.addEventListener("connect", () => {
    console.log("port is back, re-open if you want");
  });
  port.addEventListener("disconnect", () => {
    console.log("user unplugged the cable / went out of bluetooth range");
  });
}
