requireInteraction tester

Fire a real notification with requireInteraction on or off. On most platforms the flag keeps the toast on screen until dismissed. For attributed app notifications on macOS the flag is now ignored — macOS users choose temporary vs persistent banners per app in System Settings, exactly as WebKit already behaves. The log reports the real permission flow, the real constructor call and its real events.

Notifications need user permission and a real display surface. In a headless or permission-denied environment the log below will show the denied/unsupported path — that's the honest outcome, not a failure of the demo.

the call

await Notification.requestPermission();
const n = new Notification("Standup in 5 minutes", {
  body: "Daily standup starting soon",
  requireInteraction: true,   // ignored for attributed app notifications on macOS
});
n.onshow  = () => console.log("shown");
n.onclose = () => console.log("closed");
