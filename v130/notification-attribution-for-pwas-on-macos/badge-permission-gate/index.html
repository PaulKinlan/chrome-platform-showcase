Badge permission gate

The Badging API promise resolves whether or not a badge appears — that's why the macOS change is sneaky. With app attribution, the dock badge is part of the per-app notification grant: no notification permission, no badge, silently. This page makes the real navigator.setAppBadge() / clearAppBadge() calls and pairs each result with the live Notification.permission so the gate is visible.

Badges only render on an installed app's icon (dock/taskbar). In a browser tab the calls still exercise the real API surface — the log reports exactly what resolved or threw.

the gate, in code

// resolves fine either way — the OS decides whether anything renders
await navigator.setAppBadge(7);

// on macOS with attributed PWAs (Chrome 130 trial → 152 stable):
//   Notification.permission === "granted"  → badge renders
//   anything else                          → silently nothing
// so pair the two:
if (Notification.permission !== "granted") {
  await Notification.requestPermission();
}
await navigator.setAppBadge(unreadCount);
