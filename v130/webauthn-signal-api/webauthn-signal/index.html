WebAuthn Signal API simulator

Pretend you're the relying party. Add and remove credentials in the server-side list, then click "signal" — the demo invokes the real PublicKeyCredential.signalAllAcceptedCredentials / signalUnknownCredential / signalCurrentUserDetails calls so password managers can update their UI.

requires Chrome 130+ on a real RP The signal calls only do something useful when the page is a real registered RP for credentials currently stored in a password manager or platform authenticator. On other origins the calls succeed but no UI updates. The demo invokes them and reports the outcome.

server-side accepted credentials

the code

// Tell authenticators which credential IDs the RP still accepts
// — they can prune entries you've revoked.
await PublicKeyCredential.signalAllAcceptedCredentials({
  rpId: "example.com",
  userId: base64UrlDecode("dXNlci0xMjM"),
  allAcceptedCredentialIds: ["cred-1-id-b64url", "cred-2-id-b64url"],
});

// Tell authenticators a specific credential is unknown to us:
await PublicKeyCredential.signalUnknownCredential({
  rpId: "example.com",
  credentialId: "cred-orphan-b64url",
});

// Update the display name shown for this user across managers:
await PublicKeyCredential.signalCurrentUserDetails({
  rpId: "example.com",
  userId: base64UrlDecode("dXNlci0xMjM"),
  name: "alice@example.com",
  displayName: "Alice Example",
});
