Concurrent smooth scroll

Multiple scrollIntoView calls with smooth behavior can animate concurrently when their containers differ. Stops cascading scrolls from cancelling each other.

The canonical shape of the API:

el1.scrollIntoView({ behavior: "smooth" });
el2.scrollIntoView({ behavior: "smooth" });
// Both target ancestors can now animate concurrently.
