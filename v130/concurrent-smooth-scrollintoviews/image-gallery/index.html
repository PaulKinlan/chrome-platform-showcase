Gallery Scroll Sync

Click a thumbnail to scroll both the main view and the thumbnail strip simultaneously. Chrome 130 lets both scrollIntoView({behavior: 'smooth'}) calls animate at once — no more cancellation.

Mode:
Pre-130 simulation: a 20ms delay between calls causes the first scroll to be cancelled by the second.
Click a thumbnail to begin.

the code

In Chrome 130 mode, both calls are issued in the same microtask — the browser animates them concurrently:

// Chrome 130: both scroll simultaneously
thumbEl.scrollIntoView({ behavior: 'smooth', inline: 'center' });
mainEl.scrollIntoView({ behavior: 'smooth', block: 'start' });

In Pre-130 mode, a setTimeout simulates the old behaviour where the engine only tracks one smooth scroll per scroll container, so the second call interrupts the first:

// Pre-130 simulation: delay causes first animation to abort
thumbEl.scrollIntoView({ behavior: 'smooth', inline: 'center' });
setTimeout(() => {
  // By the time this runs, the thumb scroll has started.
  // In pre-130 browsers, starting a new smooth scroll on a
  // related container aborts the in-progress animation.
  mainEl.scrollIntoView({ behavior: 'smooth', block: 'start' });
}, 20);
