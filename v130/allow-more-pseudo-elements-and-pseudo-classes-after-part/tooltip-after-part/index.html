Tooltip via ::part()::after

A custom <info-card> element exposes its trigger button via part="trigger". The host page adds tooltips using info-card::part(trigger)::after { content: attr(data-tip) } — a ::after pseudo-element chained directly to ::part(). This chain was rejected by the CSS parser before Chrome 130.

✓ Chrome 130 mode — tooltips via info-card::part(trigger)::after

the code

<!-- Custom element exposes its button via part="trigger" -->
<info-card data-tip="View analytics dashboard"></info-card>

/* Host-page CSS — Chrome 130 required */

/* Hover state via pseudo-class after ::part() */
info-card::part(trigger):hover {
  background: var(--text-black);
  color: var(--bg-ivory);
}

/* Tooltip via pseudo-element after ::part() */
info-card::part(trigger)::after {
  content: attr(data-tip);
  position: absolute;
  bottom: calc(100% + 0.5rem);
  left: 50%;
  transform: translateX(-50%);
  /* ... tooltip styles ... */
  opacity: 0;
  transition: opacity 0.15s;
}
info-card::part(trigger):hover::after {
  opacity: 1;
}

Before Chrome 130, ::part(trigger)::after and ::part(trigger):hover were silently dropped by the CSS parser. The shadow-DOM component had to build its own tooltip system internally.
