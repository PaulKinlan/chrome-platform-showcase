Stable Editing

text-wrap: stable is the editing-optimised value. When text is typed or edited, stable prevents adjacent lines from reflowing — other lines stay visually stable as you type. Compare against balance and pretty which both reflow on each keystroke.

Value text-wrap-mode text-wrap-style Use case
wrap wrap auto Default. Wraps at word boundaries, no optimization.
nowrap nowrap auto Single line. Overflow or clip.
balance wrap balance Short headings, card titles. Equalises line length. Reflows on edit.
pretty wrap pretty Paragraphs. Prevents orphaned last words. Reflows on edit.
stable wrap stable Editable content. Keeps adjacent lines stable during typing. Minimal reflow. 
/* Editing context: use text-wrap: stable so adjacent
   lines don't jump as the user types */
[contenteditable] {
  text-wrap: stable;
}

/* Equivalent using longhands */
[contenteditable] {
  text-wrap-mode: wrap;
  text-wrap-style: stable;
}

/*
  text-wrap: stable is the answer to a specific UX problem:
  With balance or pretty, every keystroke can cause ALL lines
  to reflow as the algorithm re-optimises. This is distracting
  in editable contexts.

  stable only reflows the current edited line and the line
  directly affected by the change — other lines stay put.
*/
