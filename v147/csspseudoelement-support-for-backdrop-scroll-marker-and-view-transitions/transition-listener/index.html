Transition listener

View transitions expose their pseudos (::view-transition, ::view-transition-image-pair(name)) as real CSSPseudoElement objects. Script can now call .getAnimations() on each one, await per-pseudo finished, and read getComputedStyle() — instead of guessing duration from CSS or racing requestAnimationFrame.

Developer trial: CSSPseudoElement for view-transition pseudos rides behind chrome://flags/#enable-experimental-web-platform-features in Chrome 147. Falls back to document.startViewTransition().finished.

What's wired up

  1. Trigger via document.startViewTransition(...) after mutating panel text.
  2. Await transition.ready, then call document.documentElement.pseudo("::view-transition-image-pair(panel-a)") per named pseudo.
  3. For each pseudo, call .getAnimations(); for every animation, await finished.
  4. Sample getComputedStyle(pseudo) mid-transition.
const t = document.startViewTransition(() => mutate());
await t.ready;
const root = document.documentElement;
const pseudo = root.pseudo?.('::view-transition-image-pair(panel-a)');
pseudo?.getAnimations().forEach(anim => {
  anim.finished.then(() => log('image-pair finished'));
});
const cs = getComputedStyle(pseudo);
log('opacity at start:', cs.opacity);
