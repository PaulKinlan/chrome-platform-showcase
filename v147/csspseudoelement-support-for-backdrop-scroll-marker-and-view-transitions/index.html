CSSPseudoElement support for ::backdrop, ::scroll-marker and ::view-transitions

The CSSPseudoElement DOM interface — until now defined only for ::after, ::before, and ::marker — extends to ::backdrop, ::scroll-marker, and the view-transition pseudos. Authors can now attach event listeners and read computed style on these elements from JS.

concepts

  1. Backdrop Click

    A real dialog with a ::backdrop. JS gets the pseudo via dialog.pseudo("::backdrop") and adds a click listener that closes the dialog — no more click-position hit tests against the dialog rect.

  2. Transition listener

    View-transition pseudos surface as real CSSPseudoElement objects. The demo calls getAnimations() on each, awaits per-pseudo finished, and reads getComputedStyle() mid-flight.

  3. View Transition Pseudo-elements

    Trigger cross-element view transitions and watch the event log capture animationstart / animationend events on ::view-transition-old(), ::view-transition-new(), and ::view-transition-group() obtained via Element.pseudo().

  4. Pseudo-element Explorer

    Interactive gallery of all pseudo-elements now reachable via element.pseudo(). Pick a pseudo type, inspect its CSSPseudoElement interface, read getComputedStyle(), and attach live event listeners — demonstrating the full programmatic surface the Chrome 147 extension unlocks.

why it shipped

For years, attaching behaviour to ::backdrop required JavaScript hit tests against the dialog's bounding box on every document-level click. Other pseudos (scroll markers, view-transition layers) had the same problem — visible in CSS, invisible to script. Extending CSSPseudoElement to cover them closes the gap. element.pseudo("::backdrop") returns a real, listenable object; from there, the usual addEventListener and computedStyleMap APIs work.
