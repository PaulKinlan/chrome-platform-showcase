Animation Pipeline Demo

Side-by-side CSS animations: left arena animates width alone (decoupled, no forced style recalc). Right arena animates width + border-style together (coupled, forces recalc on every frame). FPS counters measure the rendering difference. Sliders control duration and box count.

Chrome 147 decouples the computed value of border-width from border-style. When both properties change together in CSS animations, older Chrome recalculated styles on every frame. With decoupling, width-only animations skip that overhead.

what's measured

Each arena tracks how many requestAnimationFrame callbacks fire per second. When the coupled side forces style recalculation (because border-style changes mid-animation), it can stall the rendering pipeline — especially with many elements. The decoupled side only recomputes geometry, not style.

On fast hardware the difference may be small at low box counts. Crank the slider to 50 boxes to see the gap widen.

the CSS

/* Decoupled — width animates independently.
   Chrome 147: getComputedStyle(el).borderWidth returns the
   author value even when border-style is none. No recalc needed. */
@keyframes widen-decoupled {
  0%   { width: 20px; }
  50%  { width: 200px; }
  100% { width: 20px; }
}

/* Coupled — style changes force a full style recalc every frame */
@keyframes widen-coupled {
  0%   { width: 20px;  border-style: none;   border-width: 0px; }
  25%  { width: 80px;  border-style: solid;  border-width: 2px; }
  50%  { width: 200px; border-style: dashed; border-width: 4px; }
  75%  { width: 80px;  border-style: dotted; border-width: 2px; }
  100% { width: 20px;  border-style: none;   border-width: 0px; }
}
