v147 · CSS · Computed Values

Width/Style Invariant Test

Chrome 147 fixes computed values for border-width, outline-width, and column-rule-width — they now reflect the author-specified value regardless of whether the corresponding *-style is none. This browser runs the full invariant test suite live.

live invariant tests

Each row sets a width and a style, then reads back getComputedStyle(). Chrome 147 passes all; older Chrome fails rows where style is none.

what changed

const el = document.createElement('div');
el.style.borderWidth = '8px';
el.style.borderStyle = 'none';
document.body.appendChild(el);

const cs = getComputedStyle(el);
// Chrome <147: cs.borderTopWidth === '0px'   // width forced to 0
// Chrome 147+:  cs.borderTopWidth === '8px'   // reflects author value
