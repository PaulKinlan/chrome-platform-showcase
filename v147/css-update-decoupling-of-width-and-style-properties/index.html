CSS update: decoupling of Width and Style properties

In Chrome 147, the computed values of border-width, outline-width, and column-rule-width now reflect the author-specified value directly, independent of the corresponding *-style property — aligning Chrome with Firefox and WebKit.

concepts

  1. Computed Value Inspector

    Live getComputedStyle() probe — set any combination of border-width and border-style and see the computed value returned by the browser before and after Chrome 147.

  2. Transition Continuity

    Demonstrates the practical impact: CSS transitions on border-width now animate through none / hidden style states without a jarring snap to zero, enabling previously impossible border appearance effects.

why it shipped

The old behaviour forced border-width, outline-width, and column-rule-width to a computed value of 0px whenever the corresponding *-style was none or hidden. This was a Chrome-only deviation — Firefox and WebKit had already returned the author-specified value. The mismatch broke CSS custom properties that read getComputedStyle() to carry border widths forward for animation or layout calculations, and made border-width transitions through none impossible. Chrome 147 aligns with the spec and the other engines by computing the width independently of the style, so authors always get back what they set.
