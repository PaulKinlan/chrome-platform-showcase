Introduction

Chrome 147 ships five CSS and JavaScript features that together enable a new class of reactive, animated, accessible interfaces — without JavaScript event-driven layouts. The typographic primitives that have been fragile or JavaScript-dependent for a decade are finally first-class platform features.

This demo uses all five in a single page: scroll-driven progress, shaped borders, auto-contrast text, scoped view transitions, and precise numerical aggregation. Each feature solves a real pain-point that every front-end developer has worked around at some point.

CSS Architecture

The border-shape property redefines the CSS box geometry itself rather than clipping the painted output. When you clip with clip-path, the border is drawn and then masked away — box-shadows fall outside the clip and become invisible, and focus outlines remain rectangular. border-shape changes where the border itself lives, so every box-model layer — background, border, shadow, outline — naturally follows the shape without workarounds.

The arrow tabs in this demo use border-shape: polygon(0% 0%, 88% 0%, 100% 50%, 88% 100%, 0% 100%). The overlap between tabs creates the chevron chain effect, and both the shadow and the 2-pixel border follow the polygon exactly.

Animation Techniques

The reading progress bar uses animation-range: scroll 0% scroll 100% on a view(block) timeline. In Chrome 147, the scroll named range maps to the full scroll extent of the container — from scrollTop = 0 to scrollTop = scrollHeight − clientHeight — not just the portion where the element intersects the viewport.

Before this feature, achieving a true full-scroll progress bar required a ScrollTimeline on the container element, which needed to be referenced from JavaScript. The scroll named range on a view timeline keeps everything in pure CSS, with the subject element providing the attachment point for the timeline.

Typography

The contrast-color() function evaluates the luminance of the passed color and returns either black or white — whichever has the higher contrast ratio. Each section header sets its background via a CSS custom property and then reads color: contrast-color(var(--section-bg)). No JavaScript checks, no hard-coded overrides per background color, no pre-vetted palette required.

As Chrome's color engine evolves, contrast-color() is expected to accept any two colors to compare — but even in its current binary form it solves the most common case: choosing readable text for a dynamically chosen or user-supplied background color.

Accessibility

Element-scoped view transitions — element.startViewTransition(callback) — scope the transition to the element's subtree. The rest of the page stays fully interactive: a user navigating with a keyboard does not have the entire document frozen while one section collapses. Multiple concurrent transitions can run on independent parts of the DOM without interfering.

Math.sumPrecise may seem unrelated to accessibility, but reading-level calculators, syllable counters, and Flesch–Kincaid readability tools all rely on accurate word and sentence counts. Floating-point accumulation errors in long documents produce unreliable estimates — Math.sumPrecise removes that variability.

featurerole in this demo

code

/* timeline named range "scroll" (Chrome 147) — full-container progress */
.progress-fill {
  animation: fill linear both;
  animation-timeline: view(block);
  animation-range: scroll 0% scroll 100%;  /* new in Chrome 147 */
}

/* CSS border-shape (Chrome 147) — box-shadow + border follow the polygon */
.nav-tab {
  border-shape: polygon(0% 0%, 88% 0%, 100% 50%, 88% 100%, 0% 100%);
  box-shadow: var(--thin-shadow);  /* shadow now follows the shape */
}

/* contrast-color() (Chrome 147) — automatic readable text */
.section-header {
  background: var(--section-bg);
  color: contrast-color(var(--section-bg));
}

/* element-scoped view transition (Chrome 147) */
const card = document.getElementById('section-card');
card.startViewTransition(() => {
  card.classList.toggle('collapsed');
});

/* Math.sumPrecise (Chrome 147) — Neumaier compensated summation */
const total = Math.sumPrecise(wordCountsArray);
