rAF Budget Monitor

Use style and layout intensity sliders to generate slow rAF-adjacent work. The monitor tracks maximum LoAF duration, split style/layout values when exposed, and fallback runs when the origin-trial fields are unavailable.

Baseline LoAF support is enough to observe long frames, but the split timing fields require the ChromeStatus origin trial. The log records the distinction for every run.
