what to look for

After running both workloads, compare the stacked bars. The style workload should produce entries where styleDuration dominates; the layout workload should show a higher layoutDuration share. If styleDuration and layoutDuration are both near zero even on slow frames, the browser you're using does not yet expose the new fields.

remediation guide

// High styleDuration → fix CSS selector complexity
// - Reduce selector specificity nesting
// - Avoid universal selectors inside component scopes
// - Prefer class selectors over attribute selectors
// - Reduce the number of rules that apply at once

// High layoutDuration → fix forced layout
// - Batch DOM writes before reads (avoid interleaving)
// - Cache geometry values instead of re-reading
// - Use CSS transforms instead of top/left
// - Reduce DOM depth in recalculated subtrees

// High forcedStyleDuration / forcedLayoutDuration
// → synchronous recalculation from JS reads after writes
//   (e.g. reading getComputedStyle() or offsetWidth
//    after writing className or style.*)
