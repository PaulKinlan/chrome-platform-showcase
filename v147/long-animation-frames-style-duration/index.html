Long Animation Frames style duration

Chrome 147 extends Long Animation Frame (LoAF) entries with styleDuration, forcedStyleDuration, layoutDuration, and forcedLayoutDuration — letting you pinpoint whether a slow frame was caused by expensive CSS recalculation, layout, or both.

Origin trial feature. Enable via chrome://flags or register for the origin trial at developer.chrome.com/origintrials.

concepts

  1. LoAF Style Duration Monitor

    Trigger intentionally long frames with heavy style work and watch the new styleDuration and forcedStyleDuration fields appear in live LoAF entries — separated from duration so you can pinpoint CSS recalculation cost.

  2. Style vs Layout Analyzer

    Run two synthetic workloads — one that triggers expensive style recalculation, another that triggers expensive layout — and compare how styleDuration vs layoutDuration differ between them.

why it shipped

Long Animation Frame Timing (Chrome 123) exposed slow frames as PerformanceLongAnimationFrameTiming entries with a total duration. But CSS performance issues have two very different root causes: expensive style calculation (matching selectors and computing values) and expensive layout (computing geometry after style). Both show up as the same opaque duration number. Without the breakdown, developers can't tell whether to fix their CSS selector complexity, reduce DOM depth, or both. styleDuration and layoutDuration solve this by splitting the frame's time into components that map directly to distinct remediation strategies.
