event lifecycle

onBeforeRequest no SecurityInfo

Fires before the request is made. No TLS information is available yet — the connection has not been established.

frame.webRequest.onBeforeRequest.addListener(
  (details) => {
    // details.url, details.method, details.requestId
    // No certificate info available at this point
  },
  { urls: ['https://*/*'] }
);
onHeadersReceived partial

Fires after server response headers arrive. The TLS handshake is complete. Request the tlsInfo extra info key to access SecurityInfo.

frame.webRequest.onHeadersReceived.addListener(
  (details) => {
    // details.tlsInfo is available with 'tlsInfo' extra info
    const si = details.tlsInfo;
    if (si) {
      console.log(si.certificate.fingerprint); // SHA-256 hex
    }
  },
  { urls: ['https://*/*'] },
  ['responseHeaders', 'tlsInfo']  // request tlsInfo
);
onCompleted full

Fires when the response body is fully received. Full SecurityInfo, including the complete certificate chain, is available here.

frame.webRequest.onCompleted.addListener(
  (details) => {
    const si = details.tlsInfo;
    if (si?.certificate) {
      const fp = si.certificate.fingerprint.sha256;
      // Compare fp against known good fingerprint
      if (fp === KNOWN_FINGERPRINT) {
        openDirectSocket(details.url);  // safe to proceed
      }
    }
  },
  { urls: ['https://*/*'] },
  ['responseHeaders', 'tlsInfo']
);

SecurityInfo object

details.tlsInfo — fields available with 'tlsInfo' extra info key
state
string
Security state: "secure", "insecure", "mixed", "broken", or "invalid".
certificate
CertificateInfo
The leaf certificate. Contains subject, issuer, validityStart, validityEnd, and fingerprint.sha256.
certificateChain
CertificateInfo[]
Full certificate chain from leaf to root CA, each with the same fields as certificate.
keyExchangeGroup
string
Key exchange group (e.g. "X25519") used in the TLS handshake.
cipher
string
Cipher suite in use (e.g. "AES_128_GCM").
protocol
string
TLS protocol version (e.g. "TLS 1.3").
