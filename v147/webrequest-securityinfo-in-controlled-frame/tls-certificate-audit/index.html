TLS Certificate Audit

Reads SecurityInfo from chrome.webRequest.onCompleted inside a ControlledFrame and grades each attribute of the TLS connection A–F. An overall score (0–100) summarises the connection's security posture.

Isolated Web App context required chrome.webRequest.SecurityInfo in ControlledFrame is only available to IWAs installed on ChromeOS. This page simulates the securityInfo object for well-known TLS configurations so you can explore the audit logic without an IWA context.

how the audit works

  1. In a real IWA, chrome.webRequest.onCompleted fires with securityInfo in the extraInfoSpec list.
  2. The securityInfo object exposes: protocol version, cipher suite, key exchange group, certificate validity, and certificate transparency status.
  3. Each attribute is graded independently. The overall score (0–100) is a weighted average.
  4. A score below 70 should prompt the IWA to warn the user or refuse a Direct Sockets connection to the same host.
// Inside an IWA's ControlledFrame
chrome.webRequest.onCompleted.addListener(
  details => {
    const info = details.securityInfo;
    console.log(info.protocol);       // "TLS 1.3"
    console.log(info.cipherSuite);    // "TLS_AES_128_GCM_SHA256"
    console.log(info.keyExchange);    // "ECDHE" | "X25519Kyber768"
    console.log(info.certificateTransparencyCompliance); // "compliant"
    const cert = info.certificates[0];
    console.log(cert.validityPeriod); // { start: ..., end: ... }
  },
  { urls: ['<all_urls>'] },
  ['responseHeaders', 'extraHeaders']
);
