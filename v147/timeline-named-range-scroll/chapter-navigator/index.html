A sticky sidebar driven by scroll-timeline named ranges. Each chapter section registers a view-timeline-name — the sidebar highlights the active chapter as it enters the viewport, using pure CSS animation-timeline with animation-range.

Chapter 1

Introduction

Scroll down to move through the chapters. The sidebar navigation highlights the current chapter using view-timeline-name: --ch1 paired with an animation-timeline on the indicator element.

Named ranges (entry, contain, exit) let you trigger the active state exactly when the section is centred in the viewport.

Chapter 2

Background

Before Chrome 147, animating based on scroll position required JavaScript scroll event listeners with getBoundingClientRect() on every frame — expensive and jank-prone. Named scroll timelines move this entirely into CSS.

Chapter 3

Approach

Each section sets view-timeline-name: --chN. The sidebar link's ::before dot listens to that named timeline with animation-timeline: --chN and animation-range: contain to drive its active appearance.

Results

The scroll-linked indicator reacts at 60 fps, composited off the main thread, without a single line of scroll JavaScript. The JS fallback using IntersectionObserver is provided for browsers that do not yet support named scroll timelines.
