Section Dots

A sticky dot navigator alongside a scrollable article — each dot uses a CSS view-timeline with the scroll named range so its activation covers the full scroll extent of the container, not just when the section enters the viewport. The progress bar and active dot update via JavaScript reading the scroll position, but the timing model is driven by the new range.

Chrome 147+ — view-timeline-name + scroll named range. In older browsers the dot navigator falls back to a JavaScript IntersectionObserver approach that tracks section visibility.

Scroll the article · the dot navigator tracks your position · click any dot to jump to that section

Scroll progress — view-timeline-name / scroll range
/* Section dots driven by the 'scroll' named range — Chrome 147 */

.article {
  view-timeline-name: --article;
  view-timeline-axis: block;
}

.progress-bar {
  animation: grow-bar linear;
  animation-timeline: --article;

  /* 'scroll' named range = full container scroll extent */
  animation-range: scroll 0% scroll 100%;
}

@keyframes grow-bar {
  from { width: 0% }
  to   { width: 100% }
}

/* Old approach: attach to the scroller (not the article),
   or use a scroll-timeline — both require more boilerplate.
   The 'scroll' named range on a view-timeline gives you
   the same result attached to the subject element itself. */
