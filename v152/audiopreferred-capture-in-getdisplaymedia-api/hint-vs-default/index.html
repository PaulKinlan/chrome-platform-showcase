Hint vs Default

Run the same capture two ways — once with audioSelection: "preferred", once without — and record what each returned. The hint's whole job is to raise the odds that audio comes along, without ever forcing it. This table is where you'd see that difference on a browser that implements the hint.

On a browser without the flag both rows behave identically — the unknown audioSelection member is ignored, so each call is just getDisplayMedia({ video, audio: true }). Enable it in Chrome 152+ via chrome://flags/#enable-experimental-web-platform-features or --enable-blink-features=GetDisplayMediaAudioSelection to exercise the real hint. Either way the picker still decides — this compares outcomes, it does not claim the hint fired.

the two calls

// A — with the preference hint
await navigator.mediaDevices.getDisplayMedia({
  video: true, audio: true, audioSelection: "preferred"
});

// B — plain request, no hint
await navigator.mediaDevices.getDisplayMedia({
  video: true, audio: true
});
