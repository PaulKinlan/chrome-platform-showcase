Space Preserved

The reason alpha() exists. Take a wide-gamut origin and fade it. alpha(from … / k) keeps the origin's own color space; the old rgb(from … r g b / calc(alpha*k)) re-materialises the color in sRGB and can clip it out of gamut. The computed-value read-back under each swatch is the proof.

the difference, in one line

:root { --p: color(display-p3 0 1 0); }   /* out-of-sRGB green */

.a { color: alpha(from var(--p) / 50%); }            /* stays display-p3 */
.b { color: rgb(from var(--p) r g b / calc(alpha*.5)); } /* clamped to sRGB green */

For a plain sRGB origin (like hsl blue) both columns match — the fallback is exact. The gap only opens on colors that live outside sRGB, which is precisely where a translucency helper must not silently degrade the color.
