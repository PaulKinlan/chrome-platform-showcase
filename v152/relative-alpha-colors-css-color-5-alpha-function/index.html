Relative Alpha Colors (CSS Color 5 alpha() function)

The new alpha() relative-color function refers to an origin color and changes only the alpha channel — alpha(from var(--c) / 80%) to replace it, or alpha(from var(--c) / calc(alpha * 0.5)) to scale it. Unlike rgb(from …), the result stays in the origin color's own color space, so an oklch() origin stays oklch().

concepts

  1. Alpha Lab

    Pick an origin color and dial its transparency two ways: replace the alpha (/ 80%) or scale the origin's own alpha with calc(alpha * k). Live swatch over a checkerboard, the exact declaration, and the browser's real getComputedStyle() read-back.

  2. Space Preserved

    Why alpha() exists. Fade a wide-gamut oklch() / color(display-p3 …) origin with alpha() versus the old rgb(from … r g b / calc(alpha*0.5)) trick. alpha() keeps the origin space; rgb(from …) flattens it to sRGB. Both computed values shown side by side.

  3. Translucent Tokens

    One brand token, a whole set of translucent surfaces. Derive a scrim, a hover tint, a focus ring and a disabled fill from a single --brand custom property with alpha(from var(--brand) / calc(alpha * k)) — change the brand color and the entire derived palette re-tints live.

why it shipped

CSS Color 4 defined relative color syntax so a color could be re-derived from an origin (rgb(from …), hsl(from …), etc.), but adjusting just transparency meant re-stating every channel in some concrete color space — which converts the origin into that space and can shift the color or clip its gamut. CSS Color 5 adds the dedicated alpha() function: the color components of the origin are left untouched, only the alpha is modified or replaced, and the result stays in the origin's color space. That makes it the correct primitive for design-system opacity scales built from a single color token.
