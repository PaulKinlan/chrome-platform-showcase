WebAudio: Configurable render quantum

The renderSizeHint option lets Web Audio contexts request default, hardware, or a numeric render quantum size, while BaseAudioContext.renderQuantumSize reports the actual frames used.

concepts

  1. AudioContext probe

    Create a real AudioContext with a selected renderSizeHint and read renderQuantumSize when exposed.

  2. Offline render probe

    Use OfflineAudioContext options to render a short silent buffer with the requested quantum size.

  3. Latency calculator

    Compare 128-frame default rendering against hardware-sized or larger render quanta at 48 kHz.

why it shipped

Fixed 128-frame Web Audio processing can mismatch platform audio blocks such as 192-frame Android buffers, creating bursty CPU load. Configurable render quantum sizes let audio apps trade latency and callback overhead more deliberately.
