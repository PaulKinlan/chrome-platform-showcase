DBSC live simulator

Device Bound Session Credentials bind a session cookie to a hardware key. The browser proves possession of that key on every refresh — a stolen cookie replayed from another device just fails. Step through the simulator below to see real key generation, signing, and verification using the WebCrypto API.

Flag DBSC is gated on chrome://flags/#device-bound-session-credentials (and rolling out as an origin trial). The simulator below uses WebCrypto's P-256 directly to show the cryptographic shape — the browser's real DBSC plumbing happens in HTTP headers.

the protocol

// Server sets up the bound session.
Sec-Session-Registration: ...; alg=ES256

// Client returns a JWK public key (private key never leaves the device).
Sec-Session-Response: { jwk, sig }

// On every refresh, the client signs the challenge with the bound key.
Sec-Session-Challenge: nonce-xyz
Sec-Session-Response: sig(nonce, jwk)

// Server verifies the signature against the registered key.
// Stolen cookie replayed elsewhere has no matching key → 401.
