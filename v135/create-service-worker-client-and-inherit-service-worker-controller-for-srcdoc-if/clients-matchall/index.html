srcdoc iframe is a real Service Worker client

Before Chrome 135, a srcdoc iframe was invisible to its parent's Service Worker — clients.matchAll() would skip it and the SW couldn't postMessage into it. The interop bug the spec called out: Resource Timing reported the iframe's URLs anyway, leaving SW-mediated caching and resource policy inconsistent. The page below renders two iframes side by side — one with src, one with srcdoc — and asks the parent how many clients the Service Worker sees.

the change

// Before: srcdoc iframe was not a SW client.
clients.matchAll().then((list) => {
  // omitted any srcdoc iframes even though parent's SW controlled them.
});

// Chrome 135: srcdoc iframes inherit the parent's SW controller AND
// count as clients. matchAll() returns them and they can be messaged.
