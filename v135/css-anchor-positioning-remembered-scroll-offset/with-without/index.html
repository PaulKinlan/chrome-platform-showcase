Side-by-side: with vs. without remembered offset

The bug report that drove this feature: anchored popovers near a scroll edge would flicker between sides every time the user scrolled past the breakpoint. Two identical scroll boxes — the only difference is position-try-options: most-scroll-position on the right. Scroll both and look for the snap.

Scroll both boxes through the anchor. The left popover snaps to the opposite side when the anchor gets close to an edge. The right one keeps its initial side because the engine remembers the scroll offset at which the popover was first laid out.

the code

.popover {
  position: absolute;
  position-anchor: --anchor;
  top: anchor(bottom);
  position-try-fallbacks: --flip-top;
  position-try-options: most-scroll-position;  /* NEW in 135 */
}
