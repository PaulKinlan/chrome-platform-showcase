why this matters

Without position-try-options: most-scroll-position, the browser re-evaluates the best fitting position on every scroll frame. At the flip point the popover jumps: below the anchor one frame, above it the next. With the property the engine locks in the initial scroll-offset at which the element was first positioned and doesn't re-evaluate until the scroll offset changes significantly.

/* Chrome 135+ — remembered offset */
.popover {
  position: absolute;
  position-anchor: --anchor;
  top: anchor(bottom);
  position-try-fallbacks: --flip-above;
  position-try-options: most-scroll-position;
}
