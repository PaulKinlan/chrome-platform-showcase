Add a list of origins and see how Chrome groups them under site-keyed (pre-135) vs origin-keyed (Chrome 135) process isolation. Origin-keyed puts each unique origin in its own process, shrinking the Spectre blast radius from site to origin granularity.

checking origin isolation…
Under site-keyed isolation, https://admin.example.com and https://api.example.com can share a process because they share the same eTLD+1 (example.com). A Spectre exploit in one could read memory from the other. Origin-keyed isolation (Chrome 135) puts each origin in its own process, so subdomains are fully separated.
