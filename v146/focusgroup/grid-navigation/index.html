Grid navigation

Composite widgets like colour pickers, emoji palettes, and date grids need both axes. focusgroup="block inline" binds left/right to inline movement and up/down to block movement — and the same DOM works in RTL because the values are logical, not visual. Flip dir on the picker to see the keymap follow.

This is the second example in the explainer: an 8×6 colour grid. Tab enters the grid, the active cell remembers, arrow keys move within, Tab exits.

attribute: focusgroup="block inline grid wrap"
