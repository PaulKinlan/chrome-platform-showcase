Paste sanitizer

The motivating editor use case: read the clipboard payload on beforeinput, cancel the default, and insert a sanitized version. Toggle the rules and paste from Word, Google Docs, or any HTML source to see the cleanup happen.

Try copying a heading, link, and image from a webpage, then pasting in here. The sanitizer keeps the structure you choose and drops the rest.

the call

editor.addEventListener("beforeinput", (e) => {
  if (e.inputType !== "insertFromPaste") return;

  // v143: dataTransfer is finally available here — so we can sanitize *before* insertion
  const html = e.dataTransfer.getData("text/html");
  const text = e.dataTransfer.getData("text/plain");

  e.preventDefault();
  const cleaned = sanitize(html || text);
  document.execCommand("insertHTML", false, cleaned);
});

why this angle

The other concept just shows what's in the event. This one is the actual reason this property was added: editors that want to act on paste content (sanitize, transform, deduplicate links) finally can, without resorting to a separate paste listener race condition.
