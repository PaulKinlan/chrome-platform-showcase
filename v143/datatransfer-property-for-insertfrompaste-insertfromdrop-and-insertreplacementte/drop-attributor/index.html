Drop attributor

Drag a research snippet from the palette into the editor. The beforeinput event fires with inputType === "insertFromDrop" and the new dataTransfer property — exposing the cited source URL. The editor reads the citation, cancels the default insert, and renders a properly attributed block quote.

This is what citation-aware editors (Notion, Obsidian, Coda) need but had to hack with dragend guesswork before: the drop type is on the input event, in spec, with the same shape as a paste.

What you're seeing

  1. Each chip is a draggable element. Its dragstart sets the text/plain and a custom application/x-attribution MIME with the cite.
  2. On drop, the editor's beforeinput fires with inputType === "insertFromDrop".
  3. Before this feature shipped, the dataTransfer was not available on beforeinput — only on the separate drop event. Now, the editor reads event.dataTransfer.getData("application/x-attribution") right from the input event, cancels the default insert, and inserts a proper <blockquote> with cite.
  4. If event.dataTransfer is null, the demo falls back to a plain insertion and logs a "miss".
editor.addEventListener("beforeinput", (e) => {
  if (e.inputType !== "insertFromDrop") return;
  const dt = e.dataTransfer;
  if (!dt) return;                              // feature not on this browser
  const text = dt.getData("text/plain");
  const cite = dt.getData("application/x-attribution");
  e.preventDefault();
  insertAttributedQuote(text, cite);
});
