mDL vs VC flows

Two real issuance flows the Chrome 143 implementation supports, side-by-side: mobile driving licence (ISO 18013-7) and W3C Verifiable Credentials over OpenID4VCI. Same navigator.credentials.create call, very different request payloads.

flag required — enable chrome://flags/#web-identity-digital-credentials-creation to actually invoke the API. This page renders the wire shapes regardless.

why this angle

The other concept builds a single request and reports support probes. This one contrasts the two actual deployed flows — US/EU mDL issuance vs the OpenID4VCI ecosystem — so issuers know which request shape matches their wallet target.
