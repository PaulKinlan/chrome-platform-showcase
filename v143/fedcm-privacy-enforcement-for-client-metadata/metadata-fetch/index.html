Metadata Fetch

Chrome now strips cookies and identifiers when fetching FedCM client metadata. The IdP can serve metadata but can't correlate the request back to a specific user or RP.

The canonical shape of the API:

# IdP receives:
#   - no cookies
#   - no client identifier
# for the metadata fetch performed by Chrome on
# behalf of the FedCM flow. Prevents IdP from
# tracking which RP the user just visited.
