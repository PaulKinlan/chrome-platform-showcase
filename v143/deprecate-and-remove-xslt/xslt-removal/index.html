XSLT Removal

Chrome is removing XSLT — the long-tail browser feature for in-browser XSL transforms. Used by a vanishingly small number of sites; security and maintenance cost no longer justifies it.

The canonical shape of the API:

# XSLT support is being removed. Sites doing
# <?xml-stylesheet ...?> server-side now need to
# pre-render the HTML output, since Chrome will
# stop applying XSL transforms in the page.
