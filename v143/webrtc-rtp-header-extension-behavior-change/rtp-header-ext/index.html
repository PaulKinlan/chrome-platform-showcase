RTP Header Extension

WebRTC's RTP header extension dispatch now strictly follows the SDP-negotiated set. Interop fix; some sites that depended on the old loose behaviour need to declare extensions explicitly.

The canonical shape of the API:

# Chrome aligns its RTP header extension handling with the
# spec: extensions are only sent if explicitly negotiated
# in SDP, and required extensions are no longer dropped.
