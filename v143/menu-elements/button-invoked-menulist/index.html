Button-invoked menulist

Not every menu lives in a menubar. The one-off dropdown — row actions in a table, a filter overlay — is just <button command="toggle-menu" commandfor="…"> pointing at a standalone <menulist>. The button gets implicit aria-haspopup=menu and a live aria-expanded state, and focus moves into the menu on open. This page invokes the real attributes and reads the real ARIA wiring back out of the DOM.

Developer trial (behind a flag). Relaunch Chrome/Canary with --enable-blink-features=MenuElements to get the real toggle behaviour (--enable-experimental-web-platform-features alone did not enable it in our testing). Without the flag this page uses a clearly-labelled JS fallback so you can still see the intended flow.
