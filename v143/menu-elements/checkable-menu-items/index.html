Checkable menu items

Menus carry state: a zoom level is a radio group, “Show toolbar” is a toggle. The proposal models both by wrapping <menuitem>s in <fieldset checkable="single"> (one checked at a time, ARIA menuitemradio) or checkable="multiple" (independent toggles, menuitemcheckbox), with defaultchecked for the initial state. Click the items and read each one's real checked value below — no simulated state when the elements are upgraded.

Developer trial (behind a flag). Relaunch Chrome/Canary with --enable-blink-features=MenuElements for real checkable behaviour. Without the flag the page runs a clearly-labelled fallback that tracks state in data-fb-checked attributes instead.

the markup

<menulist>
  <fieldset checkable="single">   <!-- ARIA menuitemradio group -->
    <menuitem>50%</menuitem>
    <menuitem defaultchecked>100%</menuitem>
  </fieldset>
</menulist>

<menulist>
  <fieldset checkable="multiple"> <!-- ARIA menuitemcheckbox group -->
    <menuitem defaultchecked>Show toolbar</menuitem>
    <menuitem>Show minimap</menuitem>
  </fieldset>
</menulist>
