Application menubar

The classic desktop pattern in four elements: a <menubar> hosting <submenu> flyouts whose <menulist>s hold <menuitem>s, separators and a disabled entry. When this browser upgrades the elements you get real popover flyouts, real menubar/menu/menuitem roles and arrow-key navigation for free — the probe below reads the actual constructors and roles, and says plainly when the markup is inert.

Developer trial (behind a flag). To run the real elements, relaunch Chrome/Canary with --enable-blink-features=MenuElements. There is no chrome://flags entry for this yet, and in our testing --enable-experimental-web-platform-features alone does not enable it — use the exact Blink feature switch.

The probe checks the exact interfaces the explainer names. If they're missing, the markup above parses as HTMLUnknownElement — it stays visible (styled by this page's CSS as inert reference markup) but has no menu behaviour, no roles and no keyboard model. That state is labelled, never faked.

the markup

<menubar>
  <submenu>
    <menuitem>File</menuitem>
    <menulist>
      <menuitem>New</menuitem>
      <submenu>
        <menuitem>Open Recent</menuitem>
        <menulist>…</menulist>
      </submenu>
      <hr>
      <menuitem>Save</menuitem>
      <menuitem disabled>Print</menuitem>
    </menulist>
  </submenu>
</menubar>
