    What changed in Chrome 143
    
    
      
      
    • nonce moved — previously nonce was a top-level field inside the identity provider object. In Chrome 143 it must be nested inside params: providers: [{ configURL, clientId, params: { nonce } }].
      • 
      
    • Error code renames — IdentityCredentialError code values were renamed for clarity: "account_hint_mismatch" → "hint_mismatch", "request_not_supported" → "not_supported", "idp_not_allowed_by_rp" → "not_allowed".
