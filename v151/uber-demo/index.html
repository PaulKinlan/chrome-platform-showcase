how the features combine

v151 · HTML attribute
autocorrect="off"

Setting autocorrect="off" on the textarea tells the OS never to autocorrect input. Without it, writing getElementById, querySelector, or any camelCase identifier silently becomes mangled by the spell-checker. Toggle the checkbox above to switch between on and off and type a method name to feel the difference.

v151 · Web Speech API
Unspoken Punctuation

Chrome 151 inserts punctuation symbols without you saying their names. Dictate "function add open paren a comma b" and the transcript reads function add(a, b — commas, periods, parentheses, and semicolons appear as glyphs. Click Dictate and try it; microphone permission is required.

v151 · WebAudio
Configurable Render Quantum

AudioContext({ renderSizeHint: "hardware" }) aligns the audio processing block with the hardware buffer so tones play back within the hardware interrupt period — no perceptible lag. Click Test Audio to hear the start beep, insert tick, and stop chime that play automatically during dictation.

code

/* ── 1. autocorrect attribute (Chrome 151) ── */
<textarea
  autocorrect="off"      /* suppress OS spell-correction */
  autocapitalize="off"  /* no auto-caps at line start */
  spellcheck="false"    /* hide red squiggles on identifiers */
></textarea>

/* ── 2. WebAudio render quantum (Chrome 151) ── */
const ctx = new AudioContext({
  latencyHint: 'interactive',
  renderSizeHint: 'hardware'  /* align to hardware buffer — lowest latency */
});

/* ── 3. Web Speech unspoken punctuation (Chrome 151) ── */
const rec = new SpeechRecognition();
rec.continuous = true;
rec.interimResults = true;
/* Chrome 151 converts "comma" → "," automatically in the transcript */
rec.onresult = (e) => {
  for (let i = e.resultIndex; i < e.results.length; i++) {
    if (e.results[i].isFinal)
      editor.value += e.results[i][0].transcript;
  }
};
