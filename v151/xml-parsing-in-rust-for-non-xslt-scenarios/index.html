XML Parsing in Rust for non-XSLT scenarios

Chrome replaced its C++ XML parser with a memory-safe Rust implementation for several common parsing scenarios (not including XSLT). This is an internal engine change that eliminates memory-related vulnerabilities — no new web-facing API.

no interactive demo

This is an internal parser rewrite. There is no new API surface, no new CSS property, no observable behavior change for developers. The only way to "test" it is to feed the browser XML documents and verify they parse correctly — which they did before, just via a less memory-safe code path. A demo would be indistinguishable from "here's an XML file, it loads."

For security researchers: the change reduces the attack surface of XML parsing by using Rust's ownership model to prevent buffer overflows and use-after-free bugs. This is invisible to web developers but significant for browser security.
