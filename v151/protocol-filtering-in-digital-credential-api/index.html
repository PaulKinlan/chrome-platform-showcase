Protocol filtering in Digital Credential API

The Digital Credentials API used to be an opaque pipe for any exchange protocol. From Chrome 151 it filters requests to a normatively-referenced set — with DigitalCredential.userAgentAllowsProtocol() letting you check support before you ask. Arbitrary protocols are removed in Chrome 160.

why it shipped

The Digital Credentials API was originally designed as an opaque transport for arbitrary exchange protocols. In November 2025 the FedID Working Group resolved to normatively reference only a specific set of protocols, so the browser can reason about what it is brokering. Chrome 151 begins deprecating unspecified presentation and issuance protocols; the arbitrary opaque pipeline is fully removed in Chrome 160. Restricting to verified protocols — OpenID4VP variants and ISO mdoc — gives the platform a stronger privacy and security threat model for identity verification, and aligns Chromium with the updated industry specification.
