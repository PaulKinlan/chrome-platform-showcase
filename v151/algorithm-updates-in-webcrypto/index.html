Algorithm Updates in WebCrypto

The Web Cryptography API gains post-quantum and modern algorithms: the NIST-standardised ML-KEM key-encapsulation mechanism and ML-DSA digital-signature scheme, plus the widely-used ChaCha20-Poly1305 AEAD cipher. New SubtleCrypto methods — encapsulateBits, encapsulateKey, decapsulateBits, decapsulateKey, getPublicKey, and a static supports() — expose them through the familiar crypto.subtle surface.

concepts

  1. ML-KEM Key Encapsulation

    Generate an ML-KEM-768 key pair, encapsulateBits a shared secret against the public key, then decapsulateBits it back with the private key — and verify byte-for-byte that both peers derived the same quantum-resistant secret.

  2. ML-DSA Post-Quantum Signatures

    Generate an ML-DSA key pair, sign a message, and verify it. Flip a byte of the message or the signature and watch verification fail — a real lattice-based signature that survives a quantum adversary.

  3. ChaCha20-Poly1305 AEAD

    Authenticated encryption with the ChaCha20-Poly1305 cipher: encrypt a message with a nonce and associated data, decrypt it, then tamper with the ciphertext, nonce, or AAD and see the Poly1305 tag reject it.

  4. Algorithm Support Probe

    The new static crypto.subtle.supports(operation, algorithm) method answers, synchronously and without a network round trip, whether the browser implements a given algorithm for a given operation. Probe a live matrix across ML-KEM, ML-DSA, ChaCha20-Poly1305, and legacy algorithms.

why it shipped

A cryptographically-relevant quantum computer would break the RSA and elliptic-curve algorithms the web relies on today, and "harvest now, decrypt later" makes that a present-tense risk for long-lived secrets. NIST standardised ML-KEM (FIPS 203) for key establishment and ML-DSA (FIPS 204) for signatures; giving browsers native, audited implementations means web apps get quantum-resistant crypto without shipping large WASM libraries. ChaCha20-Poly1305 rounds out the set with a fast, constant-time AEAD that is already the default in TLS 1.3 on devices without AES hardware.
