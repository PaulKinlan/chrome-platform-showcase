Algorithm Support Probe

The update adds a static, synchronous SubtleCrypto.supports(operation, algorithm) method. Before this, the only way to feature-detect an algorithm was to actually attempt an operation and catch the rejection — asynchronous and wasteful. Now you can branch instantly. This tool calls the real method live and builds a support matrix from what your browser reports.

SubtleCrypto.supports is not exposed in this browser. It ships with the modern-WebCrypto algorithms behind the WebCryptoPQC runtime feature. To get it:
  1. Quit Chrome, then relaunch with --enable-blink-features=WebCryptoPQC (a fresh Chrome/Canary that ships the feature), or
  2. Join the WebCryptoAdditionalAlgorithms202606 origin trial.
The matrix below still renders; every cell will show the honest result (mostly unsupported without the flag), so you can see exactly what the current browser does and doesn't have.

the code

// Static and synchronous — no promise, no try/catch needed
if (SubtleCrypto.supports?.("encapsulateBits", "ML-KEM-768")) {
  // use post-quantum key encapsulation
} else {
  // fall back to ECDH
}

// Choose the best available password KDF up front
const kdf = SubtleCrypto.supports?.("importKey", "Argon2id") ? "Argon2id" : "PBKDF2";

supports is defined as a static method on the SubtleCrypto interface object, so you call it as SubtleCrypto.supports(...), not on the crypto.subtle instance. The optional-chaining ?. makes the whole expression safely return undefined on browsers that predate the method.
