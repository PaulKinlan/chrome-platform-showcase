migration checklist

  • Replace every direct-sockets-private=* in your Permissions-Policy headers with local-network=* and loopback-network=*.
  • If you only need private (RFC 1918) access and not loopback, you can omit loopback-network.
  • If you only need loopback (127.0.0.1) access, you can omit local-network.
  • The direct-sockets directive (for Isolated Web Apps) is unchanged — keep it as-is.
  • Test with both old and new headers before cutting over in production; Chrome 151 ignores direct-sockets-private silently.
