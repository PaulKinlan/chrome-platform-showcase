Duplicate Navigation Dedup

Chrome 151 adds logic to ignore duplicate navigations that are already in-flight. When a navigation to URL X is still loading and a second navigation to the same URL X is triggered, the second is silently dropped instead of restarting the navigation cycle.

What the feature actually does

Previously, if you rapidly navigated to the same in-flight URL (e.g. via location.href, link clicks, or navigate() API), each one would restart the navigation — cancelling and re-requesting the same document. Chrome 151 deduplicates these: if the current navigation target is already loading the same URL, the duplicate is ignored.

Note: setting location.hash to the same value is a no-op in all browsers (pre-existing behavior). The Chrome 151 change is about document navigations that are still in-flight, not hash changes.

What changed in the spec

// Before Chrome 151: // navigate to /page → starts loading // navigate to /page again → cancels first, restarts // navigate to /page again → cancels second, restarts // (3 full navigation cycles, wasted bandwidth + flicker) // Chrome 151+: // navigate to /page → starts loading // navigate to /page again → IGNORED (same URL already in-flight) // navigate to /page again → IGNORED // (1 navigation cycle, no waste)
