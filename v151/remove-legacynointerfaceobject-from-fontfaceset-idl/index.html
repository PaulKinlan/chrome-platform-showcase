Remove [LegacyNoInterfaceObject] from FontFaceSet IDL

Chrome previously used [LegacyNoInterfaceObject] on FontFaceSet, hiding it as a global and deleting its constructor. This change exposes FontFaceSet globally — aligning with the CSS Font Loading spec and Firefox/Safari.

no interactive demo

This is a spec-alignment fix with no new API surface — it simply makes FontFaceSet available as a global constructor where it wasn't before. There is nothing to interact with that wasn't already possible via document.fonts; the only observable difference is typeof FontFaceSet changing from "undefined" to "function". A feature-detection probe is the sole meaningful test, and it's a one-liner, not a concept demo.

quick check

Open DevTools and run: typeof FontFaceSet — should return "function" in Chrome 151+.
