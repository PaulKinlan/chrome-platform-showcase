Prefetch activation beacon

The on-prefetch-activation response header lets a server name a telemetry endpoint. When the browser actually uses a prefetched response for navigation, it fires a credentialless HEAD beacon to that endpoint — a reliable, script-proof "this prefetch paid off" signal.

why it shipped

Measuring whether a prefetch was actually used is hard. Client-side scripts miss back/forward-cache restores, double-count on cache re-runs, and race the navigation. Because the activation beacon is dispatched by the browser's navigation stack after commit — bypassing the service worker, page scripts, and extensions — it gives an accurate, tamper-proof confirmation that a prefetched document was shown to the user. To prevent it becoming a tracking vector, the endpoint must be same-origin with the prefetched page, the request carries no credentials, and the browser only "echoes back" the URL the server itself supplied.
