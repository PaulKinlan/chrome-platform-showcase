Cross-Browser Matrix

The autocorrect attribute is finally interoperable. Here's the support matrix across engines plus a live probe that reads the IDL property in this browser and reports whether the attribute reflects, inherits, and survives DOM mutation.

Interop ship. Safari has had this since iOS 5. Firefox shipped it in 127 (June 2024). Chrome 151 closes the gap. The two-letter difference: Safari uses autocorrect="on" | "off" exactly; Chrome 151 implements the standardised HTMLElement IDL autocorrect boolean accessor as well.

live probe in this browser

code

// Read the IDL property the spec exposes.
const ok = "autocorrect" in HTMLElement.prototype;

// The attribute reflects as a boolean ("on" → true, "off" → false).
const inputOff = document.querySelector("#pOff");
console.log(inputOff.autocorrect);          // false  (autocorrect=off)
inputOff.autocorrect = true;                 // sets attribute back to "on"

// Inheritance: set on the form, override on a field.
<form autocorrect="off">
  <input name="username">                  <!-- inherits off -->
  <input name="display-name" autocorrect="on">  <!-- explicit on -->
</form>
