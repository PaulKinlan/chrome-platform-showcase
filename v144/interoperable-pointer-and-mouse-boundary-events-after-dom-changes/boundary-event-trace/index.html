Boundary Event Trace

Chrome now matches Safari and Firefox on pointer/mouse boundary events after DOM mutations under the cursor. Removes a long-standing cross-browser difference.

The canonical shape of the API:

el.addEventListener("pointerover", e => log(e.type));
el.addEventListener("pointerout", e => log(e.type));
// v144: when DOM changes under the cursor,
// pointerover/pointerout fire on the appropriate
// elements per spec — matching Safari/Firefox.
