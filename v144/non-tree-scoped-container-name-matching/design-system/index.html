Design System Container

Design systems ship a Card component with @container layout queries. Before v144 the matching was tree-scoped — the consumer had to name their container too. Now the container name resolves up the layout tree regardless of where the rule was defined.

container query support: checking…

The .card rule says @container layout (min-width: 360px). Drag the slider. The card flips between narrow and wide regardless of where in the cascade the rule lives.

the API

/* author */
.host { container-type: inline-size; container-name: layout; }

/* design-system stylesheet — different tree, same name */
@container layout (min-width: 360px) {
  .card { grid-template-columns: 1fr 1fr; }
}
/* v144: name resolves across stylesheets — no tree-scope. */
