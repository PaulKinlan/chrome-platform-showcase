vec3 Padding Trap

The most-asked WebGPU question on Stack Overflow: “Why does my vec3 uniform have wrong values?” Answer: the GPU sees a vec4. With the standard layout opt-in, Chrome 144 lets you pack uniforms without the surprise.

WebGPU: checking…

old default (std140-like)

struct { vec3 col; float weight; }

vec3 forces vec4 alignment — weight slides into byte 12 BUT only if you remember the rule. Forget it and weight lives where the GPU expects col.w.

std layout (Chrome 144 opt-in)

struct { vec3 col; float weight; }

Tight packing — matches C/C++ structs and pre-computed offsets in glTF and similar assets.

Blue = col.x, green = col.y, rose = col.z, yellow = weight, grey = padding. The opt-in spares the padding byte run.

the API

// shader (WGSL)
@group(0) @binding(0) var<uniform> u: U;

// device — opt in to std layout
const device = await adapter.requestDevice({
  requiredFeatures: ["chromium-experimental-std140"]   // illustrative
});
