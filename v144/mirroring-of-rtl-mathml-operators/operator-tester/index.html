Operator Tester

Render the same MathML expression in LTR and RTL contexts. Asymmetric operators — &sum;, &int;, &lim; — should mirror in RTL so Arabic / Hebrew readers see them facing the right way. Toggle the dir attribute.

MathML in this build: checking…

LTR (dir="ltr")

∑n=1∞ 1n2

RTL (dir="rtl")

∑n=1∞ 1n2
In Chrome 144 the &sum; operator on the right mirrors horizontally. Pre-v144 it pointed the same way in both, which is wrong for RTL scripts.

the API

<math dir="rtl">
  <mo>&sum;</mo>   <!-- now mirrored -->
  <mo>&int;</mo>   <!-- now mirrored -->
  <mo>&rarr;</mo>  <!-- already mirrored pre-v144 -->
</math>
