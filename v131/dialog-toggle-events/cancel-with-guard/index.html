cancel a close with a dirty-form guard

Like popover's ToggleEvent, dialog's beforetoggle is cancellable. Calling e.preventDefault() when newState === "closed" stops the close. Paired with a "dirty form" tracker this gives you the canonical "are you sure you want to discard?" confirmation without a single mutation observer.

how the guard works

When the user clicks cancel, beforetoggle fires with oldState === "open", newState === "closed". We check our dirty flag — if dirty, call preventDefault() and show a second confirm dialog. The close happens only if the user re-clicks discard, which sets a "skip guard" flag and re-invokes close().

the api

let dirty = false;
let skipGuard = false;
form.addEventListener("input", () => { dirty = true; });

dialog.addEventListener("beforetoggle", (e) => {
  if (e.newState === "closed" && dirty && !skipGuard) {
    e.preventDefault();      // block the close
    confirmDialog.showModal();
  }
});
