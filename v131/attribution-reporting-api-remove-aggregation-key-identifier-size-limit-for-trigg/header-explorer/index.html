What the browser validates

    
    
      
  • Sources still cap key identifiers at 25 characters. A source registration with a longer aggregation_keys member is rejected before it can be persisted to the attribution database.
    • 
      
  • Triggers no longer apply that key-name cap. The same long identifier can appear in aggregatable_trigger_data.source_keys and aggregatable_values, so operators can keep descriptive names instead of short dictionaries.
    • 
      
  • Other trigger validation still exists. The contribution in aggregatable_values must stay within the per-trigger budget; this demo lets you force that failure so the key-length change does not look like a blanket bypass.
