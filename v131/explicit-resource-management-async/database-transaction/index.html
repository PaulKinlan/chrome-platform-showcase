Database transactions with await using

The TC39 proposal's motivating use case wasn't any async resource — it was the database transaction. Acquire a connection, begin a transaction, do work, commit on success, roll back on error, always release the connection. With await using, that's three lines and one disposer.

scenario

commit (success)

All inserts succeed → the asyncDispose commits and releases the pooled connection.

scenario

throw (failure)

One insert throws → the asyncDispose rolls back and still releases the connection.

log

the pattern

async function* useTx(pool) {
  const conn = await pool.acquire();
  await conn.exec("BEGIN");
  return {
    conn,
    async [Symbol.asyncDispose]() {
      try { await conn.exec(this.committed ? "COMMIT" : "ROLLBACK"); }
      finally { pool.release(conn); }
    },
    committed: false
  };
}

async function transfer(pool, from, to, amount) {
  await using tx = await useTx(pool);
  await tx.conn.exec("UPDATE accounts SET balance = balance - ? WHERE id = ?", [amount, from]);
  await tx.conn.exec("UPDATE accounts SET balance = balance + ? WHERE id = ?", [amount, to]);
  tx.committed = true;
}
