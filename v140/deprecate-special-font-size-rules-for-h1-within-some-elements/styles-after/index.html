which to pick

A is closest to the legacy behaviour but burns the same author cycles you'd spend demoting to h2. B is the cleanest answer if your markup is being audited anyway. C plays best with the rest of the new layout primitives — container queries, scroll-driven typography — because it makes h1 a function of the layout context, not the markup nesting.
