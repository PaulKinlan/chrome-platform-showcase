Cookie Prefix

__Http- cookie name prefix — server-asserted invariants on the cookie. The browser enforces HttpOnly + Secure + Path=/ regardless of the cookie's actual attributes.

The canonical shape of the API:

Set-Cookie: __Http-Auth=abc; HttpOnly; Secure; Path=/
# __Http- prefix demands HttpOnly + Secure + Path=/
# at the network layer.
