Group vs No Group

Side-by-side. Left column uses plain :target — the URL fragment owns which link is "active", so the highlight is sticky to whatever the user last clicked. Right column adds scroll-target-group: auto — the engine tracks the visible section and updates the highlight on every scroll tick.

/* same nav markup, different container property */
.nav-none  { /* nothing */                 }
.nav-group { scroll-target-group: auto;    }

.nav-none  a:target          { ... }   /* matches the URL fragment */
.nav-group a:target-current  { ... }   /* matches the visible section */

two pseudo-classes, one source of truth

:target and :target-current answer different questions. :target is "what does the URL fragment point at?" — sticky until the next click. :target-current is "what's the user actually looking at right now?" — derived from scroll position. They can match different links at the same time. A docs site usually wants both: a permanent fragment, plus a live indicator.
