Background

Carousels and TOCs sound similar but have historically needed two different code paths. The original ::scroll-marker pseudo-element pattern works well for hand-rolled paginators but breaks when you want real anchor elements that show up in tab order and screen reader landmarks.

Each section has a fair amount of text so it is reasonably tall and the scroll is meaningful. We need enough text that the next section is not visible from the top of the first.

The proposal

The proposal: tag the container around the anchor list with scroll-target-group: auto. The engine resolves which anchor's target is in the scroll port and applies :target-current to it. Authors don't need to keep state.

Spec

Spec text in CSS Overflow Level 5. The :target-current pseudo-class only matches inside a group container — outside, it stays inert.

Compat

Behind a flag in older Chromiums; ships unflagged in Chrome 140. Polyfilled with an IntersectionObserver fallback that adds a .current class.

Future

Discussions about whether the algorithm should weight by visible area, by the topmost match, or a hybrid. The current default is "topmost in the scroll port".
