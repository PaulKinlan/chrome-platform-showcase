Container Timing API

Chrome 148 ships the Container Timing API — a PerformanceObserver extension that lets you measure when a specific DOM subtree has finished rendering, filling the gap between Largest Contentful Paint (single element) and the full page load.

concepts

  1. Timing Demo

    Mark containers with containertiming attributes and observe when they render. The demo simulates progressive loading of sections and shows the Container Timing entries as they fire.

why it shipped

Largest Contentful Paint tells you when the single largest element renders. But for complex pages — dashboards, article feeds, multi-panel apps — you need to know when a meaningful section finishes loading, not just one element. The Container Timing API adds a containertiming attribute to any element and surfaces a container-element entry via PerformanceObserver when that subtree has rendered its largest element. This lets teams build per-component performance budgets and alerts without instrumenting every child element individually.
