What an agent does

Rather than scripting clicks on an IdP's login button, an agentic browser calls navigator.credentials.get({ identity: { providers: [...] } }) — the browser handles account picking, token issuance, and mediation. The agent only sees the resulting IdentityCredential.token.

API shape

// Agentic FedCM login — agent calls this, not DOM-actuates the IdP page
const cred = await navigator.credentials.get({
  identity: {
    providers: [{
      configURL: 'https://accounts.google.com/fedcm.json',
      clientId: 'CLIENT_ID',
      nonce: crypto.randomUUID(),
    }],
    mode: 'active',   // allow without prior gesture in agentic context
  }
});

// cred.token is a signed JWT from the IdP
// No DOM actuation, no UI scripting, no brittle selectors
console.log(cred.token);  // eyJhbGci...
