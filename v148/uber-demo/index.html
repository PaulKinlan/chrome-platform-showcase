The Origin of AI Readers. Large language models changed what browsers could do locally. Chrome 148 shipped the Prompt API, giving pages access to an on-device Gemini Nano model — no server round-trips, no API keys, no privacy concerns about sending content off-device. The model runs inside a shared GPU process alongside the browser's rendering engine.

CSS Features at Work. Name-only container queries let the sidebar adapt its link colours by identity alone — no container-type, no size containment. The revert-rule keyword lets dark mode declarations undo specific overrides without re-specifying every property. And @supports at-rule(@starting-style) gates entrance animations for browsers that understand the at-rule.

Chrome 148 features at work

name-only container queries
The sidebar sets only container-name: article-sidebar — no container-type. Descendant rules query by identity: @container article-sidebar { .sidebar-link { … } }. Zero layout containment overhead.
clip text overflow on interaction
The article title has text-overflow: ellipsis. Click it — Chrome 148 automatically switches to clip so you can see and edit the full text. No JS focus/blur listeners required.
loading="lazy" on audio
The audio element carries loading="lazy" and preload="none". Chrome 148 extends lazy loading to <audio> and <video> — the resource is not fetched until the element nears the viewport.
@supports at-rule()
@supports at-rule(@starting-style) gates the streaming paragraph entrance animation. The transition + @starting-style block only runs in browsers that understand both at-rules.
revert-rule
Click revert-rule demo in the toolbar. The dark swatch's background and color are set to revert-rule, which removes the .dark-swatch overrides and falls back to the base .revert-swatch values.
extended lifetime SharedWorker
A SharedWorker created with extendedLifetime: true survives all clients disconnecting. The read-time counter in the sidebar persists across navigations because the worker stays alive between tab sessions.
Prompt API
The AI Summarize button calls window.ai.languageModel.create() if the Prompt API is available (Chrome 148+ with Gemini Nano). On unsupported browsers it falls back to a static summary. The badge shows live support status.

code

/* name-only container query (Chrome 148) — no container-type */
.sidebar { container-name: article-sidebar; }
@container article-sidebar {
  .sidebar-link { color: var(--accent-blue); }
}

/* revert-rule (Chrome 148) — undo specific cascade overrides */
.dark-swatch { background: var(--text-black); color: var(--bg-ivory); }
.dark-swatch.reverted { background: revert-rule; color: revert-rule; }

/* @supports at-rule() — gate @starting-style (Chrome 148) */
@supports at-rule(@starting-style) {
  .streamed-para { transition: opacity 0.5s ease, transform 0.5s ease; }
  @starting-style { .streamed-para { opacity: 0; transform: translateY(8px); } }
}

/* loading="lazy" on audio (Chrome 148) */
<audio loading="lazy" preload="none"></audio>

/* Extended lifetime SharedWorker (Chrome 148) */
const worker = new SharedWorker(url, { extendedLifetime: true });

/* Prompt API (Chrome 148) */
if ('ai' in window && 'languageModel' in window.ai) {
  const session = await window.ai.languageModel.create();
  const summary = await session.prompt('Summarize: …');
}
