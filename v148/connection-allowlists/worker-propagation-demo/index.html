Worker Propagation Demo

A key design goal of Connection Allowlists is that the policy propagates automatically from the document to every worker it spawns — Worker, SharedWorker, service worker scripts. A worker cannot make off-list requests even if the document could. This demo simulates the propagation: configure the allowlist for the document and watch how both document- and worker-fired requests are independently evaluated against the same policy.

// Chrome 148: Connection Allowlists — worker propagation // Document sets the allowlist via HTTP header or meta: // Connection-Allowlist: "https://api.example.com", "https://cdn.example.com" // Worker spawned by document — inherits the same allowlist const worker = new Worker('worker.js'); // Inside worker.js — subject to the same allowlist // This will succeed (in the allowlist): await fetch('https://api.example.com/data'); // This will throw NetworkError (not in the allowlist): await fetch('https://tracker.third-party.com/pixel'); // ↑ The worker cannot override or expand the document's policy
