Variable Fonts avar2 Table

Chrome 148 adds support for the OpenType avar2 table, which lets variable font designers define non-linear axis relationships — axes that influence each other — enabling far more expressive and coordinated typographic animations.

concepts

  1. Axis Explorer

    Interactive sliders for registered variable font axes — weight, width, optical size, italics, slant. Animate axes individually or all together to explore the design space that avar2 makes possible. Works with any variable font loaded via @font-face.

why it shipped

The original OpenType avar table lets font designers remap axis values non-linearly — so "weight 50" might not sit exactly halfway between thin and black, but could curve toward one end. The avar2 table extends this with inter-axis relationships: the remapping of one axis can depend on the current value of another. For example, a font could specify that as optical-size decreases, the weight axis automatically gains a small compensation to maintain apparent stroke weight at small sizes. Without avar2, browsers ignored these relationships, meaning fonts designed to use them would not animate or render as intended. Chrome 148 implements full avar2 parsing and application, unlocking the full expressive range of these fonts.
