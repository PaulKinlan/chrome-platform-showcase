Axis Explorer

Drag the sliders to move through the variable font's design space. The avar2 table lets font designers link axes together — Chrome 148 applies those relationships, so the font renders as the designer intended.

What avar2 does: When multiple axes interact (e.g. optical size adjusting stroke contrast automatically as weight changes), the font's avar2 table encodes those relationships. Chrome 148 reads and applies them — earlier Chrome versions ignored avar2 and rendered only the raw axis values, breaking fonts that depend on coordinated axes.

Variable fonts encode multiple design variations — weight, width, optical size — in a single font file. The avar2 table lets axes influence each other, producing coordinated typographic effects that a single-axis system cannot capture.

How font-variation-settings maps to avar2

/* You set raw axis values via CSS */
.text {
  font-variation-settings:
    'wght' 400,    /* weight */
    'wdth' 100,    /* width */
    'opsz' 14,     /* optical size */
    'GRAD' 0;      /* grade (custom axis) */
}

/* avar2 lives inside the font file itself.
   It remaps these values non-linearly and lets
   axes interact — Chrome 148 applies those rules.
   Your CSS stays the same; the font just renders better. */
