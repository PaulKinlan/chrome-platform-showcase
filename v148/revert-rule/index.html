revert-rule

Chrome 148 adds revert-rule — a new CSS keyword that strips the value contributed by cascade layer rules and falls back to whatever the cascade would have resolved to without any @layer rules. Fine-grained escape hatch from layered styles.

concepts

  1. Layer Inspector

    A live cascade visualiser. Set properties in three layers — defaults, theme, overrides — then apply revert-rule on any property to watch it fall back through the layer stack.

  2. Cascade Demo

    Real component styled with @layer rules. Toggle revert-rule on individual properties to selectively undo layer contributions while keeping non-layered author styles intact.

why it shipped

Before Chrome 148, escaping @layer styles required either adding an unlayered override (which sits above all layers in specificity) or using revert — which throws away all author styles and falls back to the browser's UA stylesheet. revert-rule is a middle ground: it only removes the effect of cascade layer rules. Non-layered author styles remain. This makes it practical to build component libraries and design systems where individual consumers can opt specific properties out of the layer system without fighting cascade specificity wars.
