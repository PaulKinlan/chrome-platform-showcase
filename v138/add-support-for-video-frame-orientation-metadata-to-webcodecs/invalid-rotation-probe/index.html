Invalid Rotation Probe

The VideoFrame rotation field only accepts 0, 90, 180, or 270. Any other integer value (45, -90, 1, 360) is rejected with a TypeError or the value is coerced to a valid one, depending on the spec's IDL type. This probe runs each candidate value and records the exact throw/coerce/accept behaviour in your browser.

Result will appear here.

Expected behaviour (spec)

ValueValid?Expected behaviour
0✓ validAccepted — default orientation
90✓ validAccepted — 90° clockwise
180✓ validAccepted — 180° (upside down)
270✓ validAccepted — 270° clockwise (= 90° CCW)
45✗ invalidTypeError — not in the allowed enum {0, 90, 180, 270}
-90✗ invalidTypeError — negative values not in the enum
360✗ invalidTypeError — 360 not in {0, 90, 180, 270} even though it's a full turn
1✗ invalidTypeError — non-right-angle degrees not accepted
undefined✓ coercedCoerced to 0 (omitted = default)
"90"? coercedDepends on IDL type — may coerce string "90" to 90
 
// Only 0 | 90 | 180 | 270 are valid
const valid = [0, 90, 180, 270];
const invalid = [45, -90, 360, 1, 91, 135];

for (const r of [...valid, ...invalid]) {
  try {
    const frame = new VideoFrame(canvas, { timestamp: 0, rotation: r });
    console.log('Accepted:', r, '→ frame.rotation =', frame.rotation);
    frame.close();
  } catch (e) {
    console.log('Rejected:', r, '→', e.name + ':', e.message);
  }
}
