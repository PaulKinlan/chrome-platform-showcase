Chrome 138 adds "prefetchcache" and "prerendercache" as new directives for the Clear-Site-Data response header. Previously, you could only clear "cache" (HTTP cache) or "cookies". These new directives let server-side events (e.g. a user logs out, cart changes) selectively invalidate prefetch/prerender speculation without flushing the entire HTTP cache.
