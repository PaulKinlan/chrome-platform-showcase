Temperature gauge gradient

One number — temperature in °C — drives five overlapping thresholds (cold, cool, mild, warm, hot) plus a continuous colour gradient. Each threshold is one progress() call. The clamp-to-0..1 behaviour is exactly what you want here: bands stay quiet until you cross their edges.

Unflagged in Chrome 138. progress(value, start, end) returns a unitless number — clamped to 0..1 by the engine when used in contexts that require interpolation. Used here for ranged thresholds, scroll-tied headers, and live oklch() hue interpolation.

The CSS

.gauge {
  --p:   progress(var(--t), var(--t-min), var(--t-max));   /* 0..1 across range */
  --hot: progress(var(--t), 28, 38);                       /* 0 below 28°, 1 above 38° */

  /* Continuous colour from cold to hot */
  --c: oklch(
    from black
    calc(40% + 30% * (var(--hot) + var(--cold)))
    0.18
    calc(28deg + (240deg - 28deg) * var(--cold))
  );
}

What's happening

  1. progress(value, start, end) is interpolation as a primitive — the inverse of mix(). Returns (value − start) / (end − start).
  2. Outside [start, end], the result is clamped 0..1 in contexts where that's meaningful — so you get banded thresholds for free.
  3. Combined with oklch() + the from keyword, one slider drives colour + length + opacity with a single math primitive.
  4. Five overlapping ranges (cold / cool / mild / warm / hot) each become their own meter bar — the bar fills as the temperature enters that band.
