/* CSS progress() — normalise any value range to 0..1 */ /* @property for typed custom props */ @property --value { syntax: ''; inherits: false; initial-value: 0; } .fill { width: calc(progress(var(--value), 0, 100) * 100%); } /* Colour zone: green below 60, amber 60–80, red above 80 */ .zone { background: oklch( 60% calc(progress(var(--value), 0, 100) * 0.3) /* chroma rises */ calc(140 - progress(var(--value), 0, 100) * 100) /* hue shifts green→red */ ); }
The key insight: progress(v, min, max) is a first-class CSS function that returns a unitless 0–1 number. That number can feed calc() to produce any CSS value — widths, colours, rotations, delays — without a JavaScript addEventListener('input') that reads a value and writes a style. The JS in this demo only updates --value on the element; all the visual consequences flow from CSS alone.
