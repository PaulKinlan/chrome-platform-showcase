Grid Layout Calc

Without typed arithmetic, calc() treats CSS custom properties as opaque strings — you can't subtract two <length> values or divide them to get a number. @property declarations unlock typed arithmetic: calc(var(--container-w) / var(--col-min-w)) produces a real <number>, enabling grid layouts whose column count, item height, and gaps are all computed purely from CSS.

How typed arithmetic works

Without @property — untyped (Chrome <138)
--a: 300px; --b: 100px;
calc(var(--a) - var(--b)) ✗ invalid
calc(var(--a) / var(--b)) ✗ type error
workaround: tan(atan2(var(--a), 1px)) hack
With @property — typed (Chrome 138+)
@property --a { syntax: '<length>'; … }
calc(var(--a) - var(--b)) ✓ <length>
calc(var(--a) / 1px) ✓ <number>
calc(var(--a) / var(--b)) ✓ unitless

Typed arithmetic chain

/* Register properties with explicit types */
@property --col-min-w {
  syntax: '<length>';
  inherits: true;
  initial-value: 180px;
}

@property --gap-size {
  syntax: '<length>';
  inherits: true;
  initial-value: 16px;
}

/* Typed division: <length> / <length> → <number> */
/* Without @property this was a type error */
.info-panel {
  --col-count: calc(var(--container-w) / var(--col-min-w));
  /* → a real <number> for use in counter(), rotate(), opacity()… */
}

/* Typed subtraction: <length> - <length> → <length> */
.grid-item {
  height: calc(var(--item-h) - var(--gap-size));
  /* valid because both properties declare syntax: '<length>' */
}
