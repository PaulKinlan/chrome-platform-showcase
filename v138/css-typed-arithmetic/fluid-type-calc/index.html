Fluid Type Calc

Fluid typography built on typed <length> custom properties. Chrome 138's typed arithmetic means calc(var(--max-size) - var(--min-size)) between two @property <length> values produces a valid <length> — a computation that was a syntax error with untyped properties.

Why typed properties unlock this formula

Untyped (pre-Chrome 138) — syntax error

--min: 1rem; /* unregistered = token */
--max: 2rem;
calc(var(--max) - var(--min))
/* ✗ can't subtract two tokens */

Typed (Chrome 138+) — works

@property --min { syntax: '<length>'; }
@property --max { syntax: '<length>'; }
calc(var(--max) - var(--min))
/* ✓ both are <length>, subtraction valid */

Typed arithmetic in practice: The fluid type formula is clamp(min, min + (max − min) × (vw − minVW) / (maxVW − minVW), max). In plain CSS this requires subtracting two <length> values — which means the browser must know both operands carry the same unit dimension. Registering the custom properties with syntax: '<length>' via @property gives the CSS engine the type information it needs to validate and perform the arithmetic at paint time, not just store the token string.
