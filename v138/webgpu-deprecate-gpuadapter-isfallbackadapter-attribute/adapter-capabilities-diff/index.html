Adapter Capabilities Diff

When Chrome 138 requests a GPU adapter it exposes detailed info via adapter.info — including the adapterType that replaces the removed isFallbackAdapter boolean. This page reads your adapter's live capabilities and shows the old vs new API surface side by side.

adapter.isFallbackAdapter // boolean — only tells you // hardware vs CPU software // true/false. No detail. adapter.requestDevice() // Limited to one boolean // to distinguish adapter type // REMOVED in Chrome 138+

Requesting adapter…
 
// Before Chrome 138 (DEPRECATED):
const adapter = await navigator.gpu.requestAdapter();
if (adapter.isFallbackAdapter) {  // ← removed
  console.log('software rendering');
}

// Chrome 138+ — use adapter.info instead:
const adapter = await navigator.gpu.requestAdapter();
const { adapterType, isFallbackAdapter, vendor, architecture } = adapter.info;
//      ^^^^^^^^^^^  ^^^^^^^^^^^^^^^^^^  ^^^^^^  ^^^^^^^^^^^^
//      new richer enum    still here!   new     new

console.log(adapterType);
// "discrete-gpu"  | "integrated-gpu" | "cpu" | "unknown"

// isFallbackAdapter is still available on adapter.info for compatibility
if (adapterType === 'cpu' || isFallbackAdapter) {
  console.warn('Software renderer — reduce quality settings');
}
