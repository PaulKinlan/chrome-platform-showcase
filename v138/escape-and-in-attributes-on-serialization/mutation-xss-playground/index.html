The mutation-XSS attack path: A sanitizer writes el.setAttribute('data-value', userInput), then reads back el.outerHTML to store or transmit. If the serialiser doesn't escape < and > inside the attribute value, the serialised string contains unescaped tag markup. When that string is later re-parsed by another innerHTML call — even in a sandboxed context — the parser sees the unescaped tags and instantiates them as live DOM nodes. Chrome 138 closes this by escaping < → &lt; and > → &gt; in attribute values during serialisation.
