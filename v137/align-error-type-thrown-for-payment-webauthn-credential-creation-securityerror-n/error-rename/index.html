Error Rename

WebAuthn payment credential creation now throws NotAllowedError instead of SecurityError, matching every other user-gated WebAuthn flow.

The canonical shape of the API:

try { await navigator.credentials.create({ payment: {...} }); }
catch (e) {
  if (e.name === "NotAllowedError") /* was SecurityError */
}
