Make a selection to see getComposedRanges() output.
New in Chrome 137: selection.direction returns "forward" when the anchor (start) precedes the focus (end), "backward" when the user dragged right-to-left, and "none" for collapsed or empty selections. Previously you had to compare anchorOffset and focusOffset manually — unreliable across node boundaries. getComposedRanges() adds shadow-DOM awareness.
