Chrome 137 behaviour: When an iframe becomes not-rendered (set to display:none or visibility:hidden), any playing media element inside it is automatically paused. When the iframe becomes visible again, media resumes. This matches the spec's "not rendered" media pause rule and prevents hidden iframes from silently consuming audio/video resources.
