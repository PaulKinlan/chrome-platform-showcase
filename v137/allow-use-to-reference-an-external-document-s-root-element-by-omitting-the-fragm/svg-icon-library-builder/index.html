SVG Icon Library Builder

Chrome 137 lets <use href="sprite.svg"> reference the root element of an external SVG document without a fragment identifier. Build an icon library: pick icons from the palette, place multiple instances on the canvas, and see the fragment-free <use> markup generated.

Why this matters: Previously, <use href="icons.svg"> without a #id fragment was silently ignored — you had to use <use href="icons.svg#icon-name"> and ensure every icon in the sprite had a unique id. Chrome 137 enables omitting the fragment entirely to reference the root <svg> element, simplifying single-icon external files and reducing id-collision risk in large sprite sheets.
