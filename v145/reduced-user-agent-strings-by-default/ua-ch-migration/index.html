UA-CH Migration

How to migrate from parsing navigator.userAgent or the HTTP User-Agent header to using User-Agent Client Hints — navigator.userAgentData in JavaScript and Sec-CH-UA-* request headers on the server.

signal mapping

Old signal (UA string parse) New signal (UA-CH) Entropy
Browser name navigator.userAgentData.brands / Sec-CH-UA Low — always sent
Major version brands[].version / Sec-CH-UA Low — always sent
Mobile/tablet navigator.userAgentData.mobile / Sec-CH-UA-Mobile Low — always sent
OS name (Windows/Mac/Linux) navigator.userAgentData.platform / Sec-CH-UA-Platform Low — always sent
OS version (e.g. "10.0") getHighEntropyValues(['platformVersion']) / Sec-CH-UA-Platform-Version High — requires opt-in
Full browser version getHighEntropyValues(['fullVersionList']) / Sec-CH-UA-Full-Version-List High — requires opt-in
Device model (Android) getHighEntropyValues(['model']) / Sec-CH-UA-Model High — requires opt-in
CPU bitness (64-bit) getHighEntropyValues(['bitness']) / Sec-CH-UA-Bitness High — requires opt-in

JavaScript migration

// Old: parse userAgent string (fragile, entropy-heavy)
const isChrome = /Chrome\/(\d+)/.test(navigator.userAgent);
const version = parseInt(RegExp.$1);
const isMobile = /Mobile/.test(navigator.userAgent);

// New: use userAgentData (Chrome/Edge only — check support first)
if (navigator.userAgentData) {
  const { brands, mobile, platform } = navigator.userAgentData;
  const chromeEntry = brands.find(b => b.brand === 'Google Chrome');
  const majorVersion = chromeEntry ? parseInt(chromeEntry.version) : null;

  // For high-entropy signals, request explicitly:
  const { platformVersion, fullVersionList } =
    await navigator.userAgentData.getHighEntropyValues([
      'platformVersion', 'fullVersionList'
    ]);
}

// Always provide a fallback for browsers without userAgentData:
function getBrowserInfo() {
  if (navigator.userAgentData) {
    return navigator.userAgentData; // structured, low-entropy
  }
  return { brands: [], mobile: false, platform: 'unknown' }; // safe default
}

server-side migration (HTTP headers)

# Old: receive User-Agent header (full string, high entropy)
# User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) Chrome/145.0.0.0

# New: opt-in to specific Sec-CH-UA-* headers
# Send Accept-CH from server to request additional signals:
Accept-CH: Sec-CH-UA-Platform-Version, Sec-CH-UA-Full-Version-List

# Browser then sends on next request:
# Sec-CH-UA: "Google Chrome";v="145", "Chromium";v="145", "Not-A.Brand";v="99"
# Sec-CH-UA-Mobile: ?0
# Sec-CH-UA-Platform: "Windows"
# Sec-CH-UA-Platform-Version: "15.0.0"   (only if opted-in)
# Sec-CH-UA-Full-Version-List: "Google Chrome";v="145.0.7395.51", ...

# Critical-CH makes signals available on the first request:
Critical-CH: Sec-CH-UA-Platform
