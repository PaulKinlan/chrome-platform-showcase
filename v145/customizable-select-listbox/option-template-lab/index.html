Option template lab

Drop richer markup inside <option> — colour swatches, icons, secondary text, two-line layouts — and watch a real listbox-rendered select handle them as the user types. Filter-as-you-go using the option's accessible name.

Behind a flag. Enable chrome://flags/#customizable-select. Without the flag, the select renders as the native popup; the controls still update its <option> content live so you can rehearse the markup.

Hold Ctrl/Cmd to multi-select. The custom markup persists; the value submitted is the value attribute, just like normal.

code

<!-- Chrome 145: when the select's UA renders as <listbox>, the rich
     markup inside <option> is preserved — not flattened to text. -->
<select id="palette" multiple>
  <option value="emerald">
    <span class="swatch" style="background:var(--accent-emerald)"></span>
    Emerald <span class="meta">— #2a8050</span>
  </option>
</select>
